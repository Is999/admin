# 用户标签骨架排障手册

本手册只覆盖用户标签工作流骨架的通用排障，适用于默认占位实现。

## 工作流未触发

1. 检查 `workflows.user_tag.enabled` 是否为 `true`。
2. 检查 `task_periodic` 中对应 workflow 是否启用。
3. 检查任务系统调度器和 worker 是否正常启动。

## full 工作流被锁住

full 模式会持有用户标签工作流互斥租约，保护结果表切换和同步快照刷新。若工作流已终态但租约仍存在，使用后台提供的精确释放能力按 `workflow_id` 和 `mode=full` 释放。

## 表结构缺失

运行前确认已执行：

- `internal/database/user_tag_runtime.sql`

骨架只要求通用标签结果表、临时表、同步快照表、runtime UID 表和 Kafka outbox 表存在。

## business_hook 无输出

这是默认行为。`business_hook` 是占位节点，当前返回 `Skipped=true`。默认版本只验证 DAG 能跑通，不保证产出具体标签。

## sync_kafka 被跳过

`sync_kafka` 只在 `full` 模式执行同步快照重建。`delta`、`targeted`、`recalculate` 模式看到跳过状态属于预期。
