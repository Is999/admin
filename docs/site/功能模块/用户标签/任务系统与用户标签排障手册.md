# 用户标签骨架排障手册

本手册只覆盖用户标签工作流骨架的通用排障，适用于默认占位实现。

## 工作流未触发

1. 检查 `workflows.user_tag.enabled` 是否为 `true`。
2. 检查 `task_periodic` 中对应 workflow 是否启用。
3. 检查任务系统调度器和 worker 是否正常启动。

## full 工作流被锁住

full 模式会持有用户标签工作流互斥租约，保护结果表切换和只读快照刷新。若工作流已终态但租约仍存在，使用后台提供的精确释放能力按 `workflow_id` 和 `mode=full` 释放。

## 表结构缺失

运行前确认已执行：

- `internal/database/assets/user_tag_runtime.sql`

骨架要求结果表、临时表、只读快照表、runtime UID 表、runtime checkpoint 表和事件 outbox 表存在。

## 通用阶段无输出

这是默认行为。`collect_scope`、`evaluate_tags`、`resolve_changes`、`persist_results` 是骨架节点，当前返回 `Skipped=true`。默认版本只验证 DAG 能跑通，不保证产出具体标签。

## dispatch_hooks 被跳过

`dispatch_hooks` 受 `workflows.user_tag.event_hook_enabled` 控制。未启用 hook 时跳过属于预期；启用后如果 outbox 中存在事件但未注册 hook，会返回错误并保留事件重试状态。

## 事件 outbox 长时间 running

`user_tag.event_outbox.retry_scan` 会先把超时的 `running` 事件恢复为 `retry`，再按批次重派 `pending/retry` 事件。若事件持续卡住，优先检查 hook 是否注册、最近错误和任务队列 worker 是否正常。
