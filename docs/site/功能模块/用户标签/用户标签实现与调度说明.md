# 用户标签实现与调度说明

用户标签当前是可替换的工作流骨架。代码只提供通用任务编排、运行期表管理、结果表切换和同步快照能力，不内置具体标签规则。

## 工作流拓扑

```text
prepare -> business_hook -> finalize -> sync_kafka
```

| 节点 | TaskType | 当前职责 |
| --- | --- | --- |
| `prepare` | `user_tag:prepare` | 清理当前 workflow 的运行期状态，准备结果表。 |
| `business_hook` | `user_tag:business_hook` | 占位节点，当前返回 `Skipped=true`。 |
| `finalize` | `user_tag:finalize` | 提交结果表；`full` 模式的该节点禁用全局自动重试。 |
| `sync_kafka` | `user_tag:sync_kafka` | `full` 模式重建同步快照并清理 outbox；非 `full` 模式跳过。 |

## 工作流清单

| 工作流 | 说明 |
| --- | --- |
| `user_tag.full.rebuild` | 全量骨架链路，包含结果表准备、提交和同步快照。 |
| `user_tag.delta.refresh` | 增量骨架链路，复用同一组节点。 |
| `user_tag.targeted.calc` | 指定 UID 的骨架链路，要求传入 `uids` 和 `tagTypes`。 |
| `user_tag.tag.recalculate` | 指定 `tagTypes` 的骨架链路。 |
| `user_tag.runtime.cleanup` | 清理运行期辅助数据，独立于主 DAG。 |
| `user_tag.kafka_outbox.retry_scan` | 扫描 outbox 异常行并尝试重推，独立于主 DAG。 |

## 框架边界

框架层只维护以下通用能力：

- 工作流注册与分片调度
- full 工作流互斥租约
- 运行期 UID 表
- 标签结果表、临时表、同步快照表
- Kafka outbox 表和异常扫描任务

如果后续需要接入具体标签规则，应在独立变更中补充阶段实现、表结构、测试和文档；当前文档不描述任何具体规则。

## 周期调度

运行期配置保留 `user_tag.full.rebuild`、`user_tag.delta.refresh`、`user_tag.runtime.cleanup`、`user_tag.kafka_outbox.retry_scan` 等入口。是否启用由 `task_periodic` 和 `workflows.user_tag.enabled` 控制。
