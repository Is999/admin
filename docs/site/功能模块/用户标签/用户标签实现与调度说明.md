# 用户标签实现与调度说明

用户标签当前是可替换的工作流骨架。代码只提供任务编排、运行期表管理、结果表切换、事件 outbox 和 hook 派发，不内置具体标签规则或业务清洗 SQL。

## 主工作流拓扑

```text
prepare
  -> collect_scope
  -> evaluate_tags
  -> resolve_changes
  -> persist_results
  -> finalize
  -> dispatch_hooks
```

| 节点 | TaskType | 当前职责 |
| --- | --- | --- |
| `prepare` | `user_tag:prepare` | 清理当前 workflow 的运行期状态，准备 full 临时结果表。 |
| `collect_scope` | `user_tag:collect_scope` | 收集候选 UID 范围；默认不读取具体业务来源。 |
| `evaluate_tags` | `user_tag:evaluate_tags` | 执行标签规则评估；默认不内置具体规则。 |
| `resolve_changes` | `user_tag:resolve_changes` | 比对标签得到和失去；默认不生成具体差异。 |
| `persist_results` | `user_tag:persist_results` | 写入标签结果和事件 outbox；默认不写具体结果。 |
| `finalize` | `user_tag:finalize` | full 模式提交结果表；该节点禁用全局自动重试。 |
| `dispatch_hooks` | `user_tag:dispatch_hooks` | 派发 gain/lost hook；未启用 hook 时跳过。 |

业务扩展应接入上述通用阶段，不应在骨架层新增登录、充值、投注、返水等具体业务节点。

## 工作流清单

| 工作流 | 说明 |
| --- | --- |
| `user_tag.full.rebuild` | 全量骨架链路，包含结果表准备、提交和事件 hook。 |
| `user_tag.delta.refresh` | 增量骨架链路，复用通用 DAG。 |
| `user_tag.targeted.calc` | 指定 UID 的骨架链路，要求传入 `uids` 和 `tagTypes`。 |
| `user_tag.tag.recalculate` | 指定 `tagTypes` 的骨架链路。 |
| `user_tag.runtime.cleanup` | 清理运行期辅助数据，独立于主 DAG。 |
| `user_tag.event_outbox.retry_scan` | 扫描事件 outbox 异常行并尝试重派，独立于主 DAG。 |

## 表结构

以下单表迁移 SQL 初始化用户标签承载表：

- 结果表：`internal/database/assets/user_tag_0.sql`、`internal/database/assets/user_tag_0_tmp.sql`；后续按 `result_shard_total` 使用 `CREATE TABLE LIKE user_tag_0` 扩展 `user_tag_%d`、`user_tag_%d_tmp`
- 运行期表：`internal/database/assets/user_tag_runtime_uid.sql`、`internal/database/assets/user_tag_runtime_checkpoint.sql`
- 事件表：`internal/database/assets/user_tag_event_outbox.sql`

结果表、运行期 UID 和事件 outbox 均使用 `shard_no = uid % 1024`。`default_shard_total` 只控制工作流任务拆分，`runtime_shard_total` 控制运行期索引分片，`result_shard_total` 控制 `user_tag_%d` 物理分表数量，默认 1，支持后续按 2 的幂扩展口径平滑增加。

`user_tag_event_outbox` 保存 `workflow_id`、`event_id`、`uid`、`tag_type`、`tag_source`、`action`、`shard_no`、`state`、`attempt`、`next_retry_at`、`locked_at`、`locked_by`、`last_error` 和 `dispatched_at`，用于可靠派发标签得到和失去事件。

所有新增表都有对应 GORM Model；当前骨架只维护表结构和通用清理，不落具体业务清洗逻辑。

## 周期调度

运行配置 active release 保留 `user_tag.delta.refresh`、`user_tag.runtime.cleanup`、`user_tag.event_outbox.retry_scan` 等入口。是否启用由 `task_periodic` 和 `workflows.user_tag.enabled` 控制；周期任务缺省启用，用户标签默认种子通过显式 `enabled: false` 保持关闭。
