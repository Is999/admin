# 用户标签操作手册

用户标签目前为工作流骨架。默认执行只跑通节点链路和通用表维护，不生成具体标签结果。

## 启停

- `workflows.user_tag.enabled=true`：注册用户标签插件、工作流和任务 handler。
- `workflows.user_tag.enabled=false`：跳过用户标签能力注册。

## 可用工作流

- `user_tag.full.rebuild`
- `user_tag.delta.refresh`
- `user_tag.targeted.calc`
- `user_tag.tag.recalculate`
- `user_tag.runtime.cleanup`
- `user_tag.kafka_outbox.retry_scan`

## 模式说明

| 模式 | 入口 | 参数要求 |
| --- | --- | --- |
| `full` | `user_tag.full.rebuild` | 不允许传 `uids` 或 `tagTypes`。 |
| `delta` | `user_tag.delta.refresh` | 可按默认参数运行。 |
| `targeted` | `user_tag.targeted.calc` | 必须传 `uids` 和 `tagTypes`。 |
| `recalculate` | `user_tag.tag.recalculate` | 必须传 `tagTypes`。 |

## 执行预期

主工作流默认只执行通用骨架节点：

```text
prepare -> business_hook -> finalize -> sync_kafka
```

`business_hook` 默认跳过，因此任务成功不代表已经产生具体标签。当前验收重点是工作流能被触发、节点状态能正常流转、表结构和租约逻辑可用。

## 常用检查

- 确认 `task_periodic` 中的用户标签调度是否启用。
- 确认 `user_tag_%d`、`user_tag_%d_tmp`、`user_tag_sync_%d`、`user_tag_runtime_uid`、`user_tag_kafka_outbox` 表已初始化。
- full 任务失败后，可通过工作流租约释放能力按 `workflow_id` 精确释放未释放的锁。
