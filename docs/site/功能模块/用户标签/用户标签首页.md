# 用户标签

用户标签模块当前只保留通用工作流骨架。它负责工作流注册、分片、结果表切换、只读快照、运行期候选集合、事件 outbox 和标签得失 hook，不内置登录、充值、投注等具体业务事实表。

## 当前范围

- 工作流插件：`user_tag`
- 主工作流：`full`、`delta`、`targeted`、`recalculate`
- 主 DAG：`prepare -> collect_scope -> evaluate_tags -> resolve_changes -> persist_results -> finalize -> dispatch_hooks`
- 独立维护：运行期表清理、事件 outbox 异常重派

## 默认行为

- `prepare` 清理本次 workflow 的运行期状态，并在 `full` 模式准备临时结果表。
- `collect_scope`、`evaluate_tags`、`resolve_changes`、`persist_results` 是可替换骨架阶段，默认跳过。
- `finalize` 在 `full` 模式切换结果表并刷新只读快照。
- `dispatch_hooks` 按 `user_tag_event_outbox` 派发 gain/lost 事件；未启用 hook 时默认跳过。
