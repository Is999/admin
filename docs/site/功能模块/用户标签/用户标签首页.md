# 用户标签

用户标签模块当前只保留框架骨架。它负责把工作流、分片、结果表切换、同步快照和运行期维护串起来，不内置具体标签规则。

## 当前范围

- 工作流插件：`user_tag`
- 主工作流：`full`、`delta`、`targeted`、`recalculate`
- 骨架节点：`prepare -> business_hook -> finalize -> sync_kafka`
- 独立维护：运行期表清理、Kafka outbox 异常扫描

## 默认行为

- `prepare` 清理本次 workflow 的运行期状态，并在 `full` 模式准备临时结果表。
- `business_hook` 是占位节点，当前默认跳过。
- `finalize` 在 `full` 模式切换结果表；其他模式按仓储实现处理。
- `sync_kafka` 只在 `full` 模式重建同步快照，其他模式跳过。
