# 用户标签骨架验收指南

本指南用于确认用户标签框架是否可用，不验收具体标签规则。

## 验收范围

- 插件注册：`workflows.user_tag.enabled=true` 后能注册工作流和 handler。
- 主 DAG：`prepare -> collect_scope -> evaluate_tags -> resolve_changes -> persist_results -> finalize -> dispatch_hooks` 能按依赖推进。
- 骨架节点：通用阶段默认跳过且不阻塞后续节点。
- 辅助工作流：`user_tag.runtime.cleanup`、`user_tag.event_outbox.retry_scan` 可独立触发。
- 锁与表：full 模式结束后不遗留互斥租约，运行期表结构完整。
- 事件 hook：`event_hook_enabled=true` 且存在 outbox 行时，必须有 hook 注册，否则不能把事件误标成功。

## 不在范围

- 具体标签定义。
- 具体标签计算规则。
- 具体标签结果抽样。
- 具体业务事实表维护。

## 通过标准

1. 工作流触发接口返回 `taskId`、`workflowId`、`workflowName` 和 `queue`。
2. 任务列表可按 workflow 查看节点状态。
3. 默认通用骨架节点成功跳过。
4. full 模式完成后可再次触发，不被历史租约阻塞。
