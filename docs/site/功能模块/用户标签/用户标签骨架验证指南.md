# 用户标签骨架验证指南

当前用户标签只验证框架链路，不验收具体标签规则。

## 验证前置

1. 已执行 `internal/database/assets/user_tag_runtime.sql`。
2. `workflows.user_tag.enabled=true`。
3. 任务 worker 已启动，并监听 `maintenance` 队列。

## full 链路

触发 `user_tag.full.rebuild` 后，重点确认：

1. `prepare` 能清理运行期状态并准备临时表。
2. `business_hook` 进入成功态，结果允许为 `Skipped=true`。
3. `finalize` 能提交结果表；该节点不依赖全局自动重试兜底。
4. `sync_kafka` 能刷新同步快照并释放 full 租约。

## 非 full 链路

- `delta`：复用主 DAG，默认可空跑完成。
- `targeted`：请求必须包含 `uids` 和 `tagTypes`。
- `recalculate`：请求必须包含 `tagTypes`。

## 验收结论

当前版本通过以下标准即为通过：

- 工作流实例创建成功。
- 节点状态按依赖顺序推进。
- 默认占位节点可跳过且不会阻塞 DAG。
- full 模式结束后不遗留互斥租约。
