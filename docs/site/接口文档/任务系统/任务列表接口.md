# 任务列表接口

## 1. 适用菜单

- 后台菜单：`任务列表`

## 2. 按状态查询任务列表

### 接口

```http
GET /api/tasks
```

### 查询参数

| 字段           | 类型     | 必填 | 说明                                                              |
|--------------|--------|----|-----------------------------------------------------------------|
| `queue`      | string | 是  | 队列名称                                                            |
| `state`      | string | 是  | `pending/active/scheduled/retry/archived/completed/aggregating` |
| `group`      | string | 否  | 仅 `aggregating` 时必填                                             |
| `taskId`     | string | 否  | 任务 ID 关键字，支持完整 ID 或片段匹配                                      |
| `workflowId` | string | 否  | 工作流实例 ID 筛选                                                     |
| `taskName`   | string | 否  | 任务名称关键字，支持按 `task_periodic.name` 或任务展示名包含匹配                     |
| `startTime`  | string | 否  | 任务活动时间开始，RFC3339；`scheduled` 按 `nextProcessAt` 过滤                 |
| `endTime`    | string | 否  | 任务活动时间结束，RFC3339；`scheduled` 按 `nextProcessAt` 过滤                 |
| `page`       | int    | 否  | 默认 `1`                                                          |
| `pageSize`   | int    | 否  | 默认 `20`，最大 `100`                                                |

## 3. 查询任务总览

```http
GET /api/tasks/overview?queue=maintenance
```

### 适用场景

- 任务列表页首屏在状态为空时聚合常用状态，并按任务活动时间倒序返回当前页任务。
- 未指定队列时，后端按当前可见队列做受控聚合，再返回当前页任务。

### 查询参数

| 字段                   | 类型     | 必填 | 说明                                                                         |
|----------------------|--------|----|----------------------------------------------------------------------------|
| `queue`              | string | 否  | 队列名称；为空时聚合全部可见队列                                                           |
| `state`              | string | 否  | `pending/active/scheduled/retry/archived/completed/aggregating`；为空时聚合常用状态并按任务时间倒序返回 |
| `group`              | string | 否  | 仅 `aggregating` 时必填                                                        |
| `taskId`             | string | 否  | 任务 ID 关键字，支持完整 ID 或片段匹配                                                 |
| `workflowId`         | string | 否  | 工作流实例 ID 筛选                                                                |
| `taskName`           | string | 否  | 任务名称关键字，支持按 `task_periodic.name` 或任务展示名包含匹配                                |
| `startTime`          | string | 否  | 任务活动时间开始，RFC3339；状态为空时默认聚合执行态；`scheduled` 按 `nextProcessAt` 过滤             |
| `endTime`            | string | 否  | 任务活动时间结束，RFC3339；`scheduled` 按 `nextProcessAt` 过滤                            |
| `page`               | int    | 否  | 默认 `1`                                                                     |
| `pageSize`           | int    | 否  | 默认 `20`，最大 `100`                                                           |
| `includeAggregating` | bool   | 否  | 状态为空时是否把 `aggregating` 纳入聚合范围                                             |

### 响应字段

| 字段               | 类型       | 说明                                                         |
|------------------|----------|------------------------------------------------------------|
| `queue`          | string   | 当前筛选队列；为空表示聚合全部可见队列                                        |
| `state`          | string   | 前端主动指定的状态                                                  |
| `effectiveState` | string   | 指定状态时本次真正命中的任务状态；空状态聚合时为空                              |
| `taskId`         | string   | 本次任务 ID 筛选关键字                                             |
| `taskName`       | string   | 本次任务名称筛选关键字                                                |
| `aggregateMode`  | bool     | 是否为多队列聚合视图                                                 |
| `queues`         | string[] | 本次实际参与查询的队列列表                                              |
| `stateTotals`    | object   | 各任务状态的队列级总数，用于前端展示可切换状态入口；该值不叠加 `taskName/workflowId` 二次筛选 |
| `tasks`          | array    | 当前页任务列表                                                    |
| `total`          | int      | 当前结果总数                                                     |
| `page`           | int      | 当前页码                                                       |
| `pageSize`       | int      | 当前页大小                                                      |

### 响应补充字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `startedAt` | string | 任务开始执行时间，RFC3339；仅任务已被 worker 领取后返回 |
| `durationMs` | int64 | 任务执行耗时，单位毫秒；`active` 状态表示已运行时长，终态表示本次执行总耗时 |
| `nextProcessAt` | string | 下次计划执行时间；定时、重试任务来自 Asynq，周期任务完成或归档后由当前有效 cron 配置推导 |
| `executionTrace` | object | 本次任务处理量统计摘要，包含 read/insert/update/delete/upsert/skip/error 计数和 details 明细 |

### 排序与时间范围

- 列表按任务活动时间倒序返回，便于优先看到最近计划或最近执行过的任务。
- `scheduled` 使用 `nextProcessAt` 作为排序和时间段过滤字段。
- `active/retry/archived/completed` 优先使用终态时间：`completedAt`、`lastFailedAt`；未进入终态时再回退 `startedAt`、`nextProcessAt`。
- `pending/aggregating` 仍按 `nextProcessAt` 兜底排序与过滤。
- `startTime/endTime` 为闭区间过滤；两者都不传时不过滤。

## 4. 查询单个任务详情

### 接口

```http
GET /api/tasks/:taskId
```

### 查询参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `queue` | string | 是 | 队列名称 |
| `taskId` | string | 是 | 任务 ID，放在路径参数 `:taskId` |

## 5. 让任务立即执行

### 接口

```http
POST /api/tasks/run/:taskId
```

### 请求示例

```json
{
  "queue": "maintenance"
}
```

## 6. 删除任务

### 接口

```http
DELETE /api/tasks/:taskId
```

### 请求示例

```json
{
  "queue": "maintenance"
}
```

## 7. 说明

- `queue` 必须提供，`taskId` 通过路径参数传入。
- 列表查询中的 `taskId` 是筛选参数，支持完整 ID 或片段匹配；它和详情接口路径参数 `:taskId` 互不冲突。
- `taskName` 用于排查周期任务执行情况，可直接输入 `task_periodic` 配置中的 `name`，也可以输入任务展示名片段。
- `taskName` 也会匹配任务类型、工作流名称、工作流节点、任务头和任务载荷中的同名字段；因此从调度器最近入队任务跳转时，可以用周期任务展示名或
  workflow 名称定位入口任务。
- `startTime/endTime` 在 `state` 为空时默认查询 `active/retry/archived/completed`，用于排查某个时间段实际执行过的任务；查询计划任务需显式选择
  `scheduled`。
- 定时任务时间段查询建议同时指定 `state=scheduled`，后端会按 `nextProcessAt` 倒序返回命中的任务。
- `stateTotals` 来自队列快照，只用于提示 `pending/active/scheduled/retry/archived/completed/aggregating`
  等状态是否有数据；真实列表仍以当前 `state/taskName/workflowId` 查询结果为准。
- Asynq Inspector 没有任务 ID、任务名称和工作流 ID 二级索引；使用 `taskId`、`taskName` 或 `workflowId` 筛选时，后端会按队列和状态做受控分页扫描。生产环境建议同时指定
  `queue`、`state`，避免在高积压队列上做过宽查询。
- `run` 常用于把 `scheduled` 任务立即拉起执行。
- `DELETE /api/tasks/:taskId` 用于清理失效任务，不建议删除仍在执行中的任务。
- 固定路径 `/api/tasks/overview`、`/api/tasks/queues`、`/api/tasks/registry/*`、`/api/tasks/config-reload`、`/api/tasks/config-reload/items` 必须优先于
  `/api/tasks/:taskId` 注册，避免固定菜单接口被误匹配为任务详情。

## 8. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 否 |  | 查询接口返回任务详情或分页数据；操作接口可为空 |

### 列表 data 内部字段

| 字段    | 类型    | 是否必须 | 默认值 | 字段解释    |
|-------|-------|------|-----|---------|
| tasks | array | 是    | []  | 当前页任务列表 |
| total | int   | 是    | 0   | 任务总数    |
