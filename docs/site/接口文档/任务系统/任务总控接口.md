# 任务总控接口

## 1. 适用菜单

- 后台菜单：`任务总控`

## 2. 手动投递通用任务

### 接口

```http
POST /api/tasks
```

### 适用场景

- 运维手工补投已注册任务。
- 管理动作较轻，不需要走完整工作流。

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `taskType` | string | 是 | 已注册任务类型 |
| `payload` | object | 是 | 任务负载 JSON |
| `queue` | string | 否 | 指定队列 |
| `group` | string | 否 | 聚合组 |
| `retry` | int | 否 | 覆盖重试次数，范围 `0-25` |
| `timeoutSeconds` | int | 否 | 覆盖超时时间，范围 `1-86400` 秒 |
| `processAt` | string | 否 | 指定执行时间 |
| `processInSeconds` | int | 否 | 延迟执行秒数 |
| `deadline` | string | 否 | 截止时间 |
| `uniqueTTLSeconds` | int | 否 | 去重 TTL |

### 请求示例

```json
{
  "taskType": "cache:refresh",
  "payload": {
    "target": "admin_menu"
  },
  "queue": "maintenance",
  "retry": 3,
  "timeoutSeconds": 300
}
```

## 3. 手动触发工作流

### 接口

```http
POST /api/tasks/workflows
```

### 适用场景

- 手工触发已注册工作流。
- 做延迟工作流、定时工作流或幂等触发。

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 工作流名称 |
| `targets` | string[] | 否 | 工作流目标参数 |
| `queue` | string | 否 | 指定队列 |
| `shardTotal` | int | 否 | 分片总数 |
| `grayPercent` | int | 否 | 灰度比例，默认 `100` |
| `uniqueKey` | string | 否 | 幂等去重键 |
| `uniqueTTLSeconds` | int | 否 | 去重 TTL |
| `retry` | int | 否 | 重试次数，范围 `0-25` |
| `timeoutSeconds` | int | 否 | 超时时间，范围 `1-86400` 秒 |
| `processAt` | string | 否 | 指定触发时间 |
| `processInSeconds` | int | 否 | 延迟触发秒数 |
| `deadline` | string | 否 | 截止时间 |

### 请求示例

```json
{
  "name": "cache.refresh",
  "targets": [
    "cache_key:demo"
  ],
  "queue": "maintenance",
  "shardTotal": 2,
  "uniqueKey": "cache:refresh:manual:20260430:01",
  "uniqueTTLSeconds": 1800,
  "timeoutSeconds": 1800
}
```

### 成功响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "触发成功",
  "data": {
    "taskId": "task_xxx",
    "workflowId": "wf_xxx",
    "workflowName": "cache.refresh",
    "queue": "maintenance"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 工作流触发结果 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| taskId | string | 是 |  | 任务 ID |
| workflowId | string | 是 |  | 工作流实例 ID |
| workflowName | string | 是 |  | 工作流名称 |
| queue | string | 是 |  | 任务队列 |

## 4. 使用建议

- 有明确模块语义时，优先用模块自己的入口，不要让使用方直接操作通用工作流接口。
- 需要补跑时，建议显式填写新的 `uniqueKey`，避免被旧任务去重。
