# 任务注册表接口

---

## 1. 查询任务类型注册表

### 请求示例

```http
GET /api/tasks/registry/task-types
```

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "taskType": "workflow:trigger",
        "description": "触发工作流",
        "queue": "maintenance",
        "registered": true
      }
    ]
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 任务类型注册表 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| list | array | 是 | [] | 任务类型列表 |

#### list[] 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| taskType | string | 是 |  | 任务类型 |
| description | string | 否 |  | 中文说明 |
| queue | string | 否 |  | 默认队列 |
| registered | bool | 是 | false | 是否已注册处理器 |

## 2. 查询工作流注册表

### 请求示例

```http
GET /api/tasks/registry/workflows
```

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "name": "cache.refresh",
        "description": "缓存刷新工作流",
        "nodeCount": 2,
        "registered": true
      }
    ]
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 工作流注册表 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| list | array | 是 | [] | 工作流列表 |

#### list[] 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| name | string | 是 |  | 工作流名称 |
| description | string | 否 |  | 中文说明 |
| nodeCount | int | 是 | 0 | 工作流节点数量 |
| registered | bool | 是 | false | 是否已注册定义 |
