# 任务系统接口总览

## 1. 文档说明

任务系统接口已按后台菜单拆分，避免把“任务总控、任务队列、任务列表、任务监控”全部写在同一份文档里。

## 2. 后台菜单对应关系

| 后台菜单 | 说明 | 文档 |
| --- | --- | --- |
| Collector任务 | 通用收集器任务列表、手动执行、死信重试 | [收集器任务接口](接口文档/任务系统/收集器任务接口.md) |
| 任务总控 | 手动投递任务、手动触发工作流 | [任务总控接口](接口文档/任务系统/任务总控接口.md) |
| 任务队列 | 查看队列、暂停队列、恢复队列 | [任务队列接口](接口文档/任务系统/任务队列接口.md) |
| 任务列表 | 查看任务、立即执行、删除任务 | [任务列表接口](接口文档/任务系统/任务列表接口.md) |
| 任务监控 | 工作流状态、配置热加载状态、运行态配置项查询 | [任务监控接口](接口文档/任务系统/任务监控接口.md) |
| 任务注册表 | 查询任务类型和工作流注册信息 | [任务注册表接口](接口文档/任务系统/任务注册表接口.md) |
| 运行配置 | 管理 `task_periodic` 和 `archive_jobs` 草稿、发布、回滚、导入 | [运行配置管理接口](接口文档/任务系统/运行配置管理接口.md) |

## 3. 统一返回结构

所有接口统一返回：

```json
{
  "status": true,
  "code": 1,
  "message": "成功",
  "data": {}
}
```

## 4. 公共约束

- `processAt`、`deadline` 必须为 RFC3339 时间格式。
- `processAt` 和 `processInSeconds` 不能同时设置。
- `uniqueTTLSeconds`、`timeoutSeconds` 必须大于 `0`。
- 任务系统未启用时，接口会返回任务系统未开启类错误。

## 5. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 任务接口返回数据，不同接口结构不同 |
