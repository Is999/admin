# 任务队列接口

## 1. 适用菜单

- 后台菜单：`任务队列`

## 2. 查询任务队列、Worker 与调度器概览

### 接口

```http
GET /api/tasks/queues
```

### 用途

- 查看每个队列的堆积、处理中、调度中、完成数。
- 查看在线 worker 的并发和权重。
- 查看任务控制台新增的调度器运行状态快照。

### 响应结构

响应 `data` 包含三组对象：

- `queues[]`：每个逻辑队列的运行快照。
- `servers[]`：当前在线 Worker 节点。
- `scheduler`：当前进程看到的周期调度器状态。

### `queues[]` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `name` | string | 队列名称 |
| `paused` | bool | 是否已暂停消费 |
| `size` | int | 队列总任务数 |
| `pending` | int | 待执行任务数 |
| `active` | int | 执行中任务数 |
| `scheduled` | int | 定时任务数 |
| `retry` | int | 重试任务数 |
| `archived` | int | 归档任务数 |
| `completed` | int | 已完成保留任务数 |
| `aggregating` | int | 聚合中的任务数 |
| `processed` | int | 当日已处理任务数 |
| `failed` | int | 当日失败任务数 |
| `latencyMs` | int64 | 队列延迟，单位毫秒 |
| `memoryUsage` | int64 | 预估内存占用，单位字节 |

### `servers[]` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | Worker 实例 ID |
| `host` | string | Worker 所在主机 |
| `pid` | int | 进程 ID |
| `status` | string | Worker 状态 |
| `concurrency` | int | 并发度 |
| `strictPriority` | bool | 是否启用严格优先级 |
| `queues` | object | 队列权重配置 |
| `startedAt` | string | 启动时间，RFC3339 |

### `scheduler` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `enabled` | bool | 配置中是否启用调度器 |
| `running` | bool | 当前进程是否已启动 leader 竞争循环 |
| `hasLeader` | bool | 当前进程是否持有 leader |
| `instanceId` | string | 当前进程实例 ID |
| `leaderLockKey` | string | leader 锁 key |
| `leaseTtlSeconds` | int | leader 锁租约时长 |
| `renewIntervalSeconds` | int | leader 锁续租间隔 |
| `syncIntervalSeconds` | int | 周期配置同步间隔 |
| `heartbeatIntervalSeconds` | int | 调度器心跳间隔 |
| `periodicTaskCount` | int | 当前有效周期任务数量 |
| `lastStatus` | string | 最近一次总体状态：`disabled/idle/waiting_leader/leader/stopped` |
| `lastMessage` | string | 最近一次总体状态说明 |
| `lastSyncStatus` | string | 最近一次同步结果：`success/failed` |
| `lastSyncMessage` | string | 最近一次同步结果说明 |
| `lastEnqueueTaskName` | string | 最近一次成功入队的周期任务名称 |
| `lastEnqueueTaskType` | string | 最近一次成功入队的周期任务类型 |
| `lastEnqueueErrorMessage` | string | 最近一次周期任务入队失败原因 |
| `lastStartedAt` | string | 最近一次启动 leader 竞争时间，RFC3339 |
| `lastHeartbeatAt` | string | 最近一次心跳时间，RFC3339 |
| `lastAcquireAt` | string | 最近一次获取 leader 时间，RFC3339 |
| `lastReleaseAt` | string | 最近一次释放或丢失 leader 时间，RFC3339 |
| `lastSyncAt` | string | 最近一次同步周期配置时间，RFC3339 |
| `lastEnqueueAt` | string | 最近一次成功入队时间，RFC3339 |
| `lastEnqueueErrorAt` | string | 最近一次入队失败时间，RFC3339 |

## 3. 暂停队列消费

### 接口

```http
POST /api/tasks/queues/pause/:queue
```

### 请求示例

```http
POST /api/tasks/queues/pause/maintenance
```

## 4. 恢复队列消费

### 接口

```http
POST /api/tasks/queues/resume/:queue
```

### 请求示例

```http
POST /api/tasks/queues/resume/maintenance
```

## 5. 说明

- `queue` 不能为空，并通过路径参数传入。
- 暂停队列只会停止消费，不会删除已存在的任务。
- 任务控制台页面里的“调度器状态总览 / 参数快照 / 时间轴”都来自 `GET /api/tasks/queues` 返回的 `data.scheduler`，不需要额外请求单独的 scheduler 接口。
- 排障结束后记得恢复队列，否则任务会持续堆积。

## 6. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 否 |  | 队列查询返回队列概览；暂停/恢复操作可为空 |

### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| queues | array | 是 | [] | 队列运行快照 |
| servers | array | 是 | [] | Worker 节点列表 |
| scheduler | object | 否 |  | 周期调度器状态 |
