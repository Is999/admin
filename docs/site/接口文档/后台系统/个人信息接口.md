# 个人信息管理接口

前端个人信息页面使用个人中心安全接口，敏感操作需要先通过 `/api/profile/check-mfa` 获取 `twoStepKey`、`twoStepValue` 后提交。

---

## 1. 查询当前用户

### 请求示例

```http
GET /api/profile
Authorization: Bearer <token>
```

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "email": "admin@example.com",
    "phone": "13800000000",
    "avatar": "",
    "mfaStatus": 1,
    "roles": []
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 当前管理员信息 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 管理员 ID |
| username | string | 是 |  | 登录用户名 |
| realName | string | 否 |  | 真实姓名 |
| email | string | 否 |  | 邮箱 |
| phone | string | 否 |  | 手机号 |
| avatar | string | 否 |  | 头像地址 |
| mfaStatus | int | 是 | 0 | MFA 状态，1 启用，0 禁用 |
| roles | array | 否 | [] | 当前管理员角色 |

---

## 2. 校验 MFA 动态码

### 请求示例

```http
POST /api/profile/check-mfa
Content-Type: application/json

{
  "secure": "123456",
  "scenario": 1
}
```

### 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| secure | string | 是 |  | MFA 动态码 |
| scenario | int | 否 | 0 | 校验场景值；修改 MFA 秘钥使用 `3` |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "校验成功",
  "data": {
    "twoStepKey": "ticket_xxx",
    "twoStepValue": "proof_xxx",
    "expireAt": "2026-05-11T12:00:00+08:00"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 二次校验票据 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| twoStepKey | string | 是 |  | 二次校验票据 Key |
| twoStepValue | string | 是 |  | 二次校验票据值 |
| expireAt | string | 否 |  | 票据过期时间 |

## 7. 修改 MFA 秘钥

修改 MFA 秘钥使用 `MFAScenarioSecret = 3`，不能复用秘钥管理场景 `11` 的二次票据。

### 请求示例

```http
PATCH /api/profile/mfa-secret
Content-Type: application/json

{
  "mfaSecureKey": "base32-secret",
  "secure": "123456",
  "twoStepKey": "ticket_xxx",
  "twoStepValue": "proof_xxx"
}
```

### 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| mfaSecureKey | string | 是 |  | 新 MFA 秘钥 |
| secure | string | 是 |  | 新秘钥生成的动态码 |
| twoStepKey | string | 条件必填 |  | MFA 二次票据 Key |
| twoStepValue | string | 条件必填 |  | MFA 二次票据值 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "修改成功"
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
