# 前台用户管理接口

admin 后台直接使用默认主库管理 `user` 表，不通过 API 代理 CRUD，也不要求单独配置前台用户库。只有资料缓存失效、登录态失效和 API 配置热加载这类必须在 API 进程内完成的运行态能力，才调用 API 内网运维接口。

## 配置边界

| 配置项 | 说明 |
| --- | --- |
| 顶层 `mysql` | `user` 表所在默认主库，前台用户管理与 admin 表共用该连接 |
| `api_service.internal_base_url` | API 内网地址，仅 admin 后端调用 |
| `api_service.ops_token` | 对应 API `ops.config_reload_token`，用于 `X-Ops-Token` |
| `api_service.timeout_seconds` | API 内网调用超时，默认 5 秒，最大 30 秒 |

## 接口总览

| 方法 | 路径 | 权限码 | 说明 |
| --- | --- | --- | --- |
| `GET` | `/api/users` | `user.list` | 查询前台用户列表 |
| `GET` | `/api/users/:id` | `user.info` | 查询前台用户详情 |
| `POST` | `/api/users` | `user.add` | 新增前台用户 |
| `PATCH` | `/api/users/:id` | `user.update` | 编辑前台用户资料 |
| `PATCH` | `/api/users/status/:id` | `user.status.update` | 启用或禁用前台用户 |
| `POST` | `/api/users/password/reset/:id` | `user.password.reset` | 重置前台用户密码 |
| `POST` | `/api/users/runtime-sync/:id` | `user.runtime.sync` | 手动同步用户资料缓存或登录态 |
| `GET` | `/api/api-runtime/config-reload` | `api_runtime.config_reload.status` | 查询 API 配置热加载状态 |
| `GET` | `/api/api-runtime/config-reload/items` | `api_runtime.config_reload.items` | 查询 API 运行态配置项 |
| `POST` | `/api/api-runtime/config-reload` | `api_runtime.config_reload.run` | 手动触发 API 配置热加载 |

## 列表查询

`GET /api/users`

查询只支持 `id`、`shardNo`、`status` 精确筛选，以及 `username`、`email`、`phone` 左前缀筛选，避免昵称模糊查询造成全表扫描。

| 字段 | 类型 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| id | int | 否 | 用户 ID |
| shardNo | int | 否 | 用户取模分片，范围 `0-999` |
| username | string | 否 | 用户名前缀 |
| email | string | 否 | 邮箱前缀 |
| phone | string | 否 | 手机号前缀 |
| status | int | 否 | `1` 正常，`0` 禁用 |

列表与详情响应中的 `id` 为雪花 ID，`shardNo = id % 1000`，后续按 shardNo 扩展分表或分片查询时无需改变接口字段。

## 写操作安全

- 新增、编辑、状态修改、重置密码、手动同步和 API 热加载均要求 MFA 二次票据。
- 前台用户管理使用 `MFAScenarioUserManage = 13`。
- API 热加载使用 `MFAScenarioAPIRuntimeManage = 14`。
- 密码使用 API 现有 bcrypt 规则，不复用 admin 管理员密码加盐规则。
- 编辑资料、禁用用户、重置密码后，admin 会调用 API 内网接口同步资料缓存或失效登录态；如果 API 内网配置缺失，写操作会拒绝执行。

## 新增请求示例

```json
{
  "username": "demo_user",
  "password": "demoPass123",
  "nickname": "Demo",
  "email": "demo@example.com",
  "phone": "",
  "status": 1,
  "twoStepKey": "ticket-key",
  "twoStepValue": "ticket-value"
}
```

```json
  "profile": true,
  "sessions": true,
  "twoStepKey": "ticket-key",
  "twoStepValue": "ticket-value"
}
```

## API 运行态配置项查询

`GET /api/api-runtime/config-reload/items`

该接口由 admin 后端代理 API 内网运维接口，前端不直接访问 API 服务。查询参数与 API 内网接口一致：

| 字段 | 类型 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| keyword | string | 否 | 配置路径、类型或已脱敏展示值关键字，最大 128 字符 |
| sensitiveOnly | bool | 否 | 是否只返回敏感配置项 |
| page | int | 否 | 页码，从 1 开始 |
| pageSize | int | 否 | 每页条数，最大 100 |

### 响应说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| connected | bool | 是否已配置并成功访问 API 内网接口 |
| items | object | API 返回的运行态配置项快照；字段结构同 API `GET /internal/system/config-reload/items` |
| message | string | 调用结果说明 |
