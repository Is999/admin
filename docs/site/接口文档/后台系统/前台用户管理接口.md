# 前台用户管理接口

admin 后台直接连接 API 用户表所在数据库管理 `api_user`，不通过 API 代理 CRUD。只有资料缓存失效、登录态失效和 API 配置热加载这类必须在 API 进程内完成的运行态能力，才调用 API 内网运维接口。

## 配置边界

| 配置项 | 说明 |
| --- | --- |
| `site_mysql.api` | 前台用户表所在数据库连接；未配置且 `api_service.user_database=api` 时，用户管理接口会返回数据库配置错误 |
| `api_service.user_database` | 前台用户表使用的命名库，默认 `api` |
| `api_service.internal_base_url` | API 内网地址，仅 admin 后端调用 |
| `api_service.ops_token` | 对应 API `ops.config_reload_token`，用于 `X-Ops-Token` |
| `api_service.timeout_seconds` | API 内网调用超时，默认 5 秒，最大 30 秒 |

## 接口总览

| 方法 | 路径 | 权限码 | 说明 |
| --- | --- | --- | --- |
| `GET` | `/api/api-users` | `api_user.list` | 查询前台用户列表 |
| `GET` | `/api/api-users/:id` | `api_user.info` | 查询前台用户详情 |
| `POST` | `/api/api-users` | `api_user.add` | 新增前台用户 |
| `PATCH` | `/api/api-users/:id` | `api_user.update` | 编辑前台用户资料 |
| `PATCH` | `/api/api-users/status/:id` | `api_user.status.update` | 启用或禁用前台用户 |
| `POST` | `/api/api-users/password/reset/:id` | `api_user.password.reset` | 重置前台用户密码 |
| `POST` | `/api/api-users/runtime-sync/:id` | `api_user.runtime.sync` | 手动同步用户资料缓存或登录态 |
| `GET` | `/api/api-runtime/config-reload` | `api_runtime.config_reload.status` | 查询 API 配置热加载状态 |
| `POST` | `/api/api-runtime/config-reload` | `api_runtime.config_reload.run` | 手动触发 API 配置热加载 |

## 列表查询

`GET /api/api-users`

查询只支持 `id` 精确筛选、`status` 精确筛选，以及 `username`、`email`、`phone` 左前缀筛选，避免昵称模糊查询造成全表扫描。

| 字段 | 类型 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| id | int | 否 | 用户 ID |
| username | string | 否 | 用户名前缀 |
| email | string | 否 | 邮箱前缀 |
| phone | string | 否 | 手机号前缀 |
| status | int | 否 | `1` 正常，`0` 禁用 |

## 写操作安全

- 新增、编辑、状态修改、重置密码、手动同步和 API 热加载均要求 MFA 二次票据。
- 前台用户管理使用 `MFAScenarioAPIUserManage = 13`。
- API 热加载使用 `MFAScenarioAPIRuntimeManage = 14`。
- 密码使用 API 现有 bcrypt 规则，不复用 admin 管理员密码加盐规则。
- 编辑资料、禁用用户、重置密码后，admin 会调用 API 内网接口同步资料缓存或失效登录态；如果 API 内网配置缺失，写操作会拒绝执行。

## 新增请求示例

```json
{
  "username": "demo_user",
  "password": "demoPass123",
  "nickname": "Demo",
  "email": "demo@example.com",
  "phone": "",
  "status": 1,
  "twoStepKey": "ticket-key",
  "twoStepValue": "ticket-value"
}
```

## 运行态同步请求示例

```json
{
  "profile": true,
  "sessions": true,
  "twoStepKey": "ticket-key",
  "twoStepValue": "ticket-value"
}
```

## API 热加载请求示例

```json
{
  "twoStepKey": "ticket-key",
  "twoStepValue": "ticket-value"
}
```
