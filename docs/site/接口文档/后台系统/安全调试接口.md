# 安全调试接口

安全调试接口用于本地和联调环境验证签名、验签、加密、解密链路。生产环境建议仅对受信运维角色开放。

---

## 1. 生成签名

### 请求示例

```http
POST /api/security/debug/sign
Content-Type: application/json

{
  "appId": "YWRtaW4wMDAx",
  "signatureType": "A",
  "signFields": ["username", "password", "secureCode"],
  "payloadText": "{\"username\":\"admin\",\"password\":\"123456\",\"secureCode\":\"abcd\"}",
  "requestId": "trace-001",
  "timestamp": "1700000000"
}
```

### 字段说明

| 字段            | 类型       | 是否必须 | 默认值   | 字段解释                                                         |
|---------------|----------|------|-------|--------------------------------------------------------------|
| appId         | string   | 是    |       | 应用 ID，和 `X-App-Id` 语义一致                                      |
| payloadText   | string   | 是    |       | 待签名 JSON 对象文本                                                |
| requestId     | string   | 否    | 自动生成  | 请求链路 ID，参与签名                                                 |
| timestamp     | string   | 否    | 当前秒级时间 | 秒级 Unix 时间戳，参与签名                                             |
| signatureType | string   | 否    | R     | 签名方式：`A` AES、`R` RSA                                         |
| signFields    | string[] | 否    | ["*"] | 参与签名字段，`*` 表示全部首层字段                                          |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "操作成功",
  "data": {
    "appId": "1",
    "requestId": "trace-001",
    "traceId": "trace-001",
    "timestamp": "1700000000",
    "signatureType": "A",
    "signFields": ["username", "password", "secureCode"],
    "payload": {
      "username": "admin",
      "password": "123456",
      "secureCode": "abcd"
    },
    "payloadText": "{\n  \"username\": \"admin\",\n  \"password\": \"123456\",\n  \"secureCode\": \"abcd\"\n}",
    "signText": "password=123456&secureCode=abcd&username=admin&key=<md5(appId-traceId-timestamp)>",
    "sign": "BASE64_AES_SIGNATURE"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 签名结果 |

#### data 内部字段

| 字段            | 类型       | 是否必须 | 默认值 | 字段解释                                   |
|---------------|----------|------|-----|----------------------------------------|
| appId         | string   | 是    |     | 实际参与签名的 AppID                          |
| requestId     | string   | 是    |     | 实际参与签名的请求标识                            |
| traceId       | string   | 是    |     | 与 `requestId` 等值，便于前端按 `X-Trace-Id` 回填 |
| timestamp     | string   | 是    |     | 实际参与签名的秒级 Unix 时间戳                    |
| signatureType | string   | 是    |     | 实际使用的签名方式                              |
| signFields    | string[] | 是    |     | 实际参与签名的字段列表                            |
| payload       | object   | 是    |     | 归一化后的待签名对象                             |
| payloadText   | string   | 是    |     | 归一化后的 JSON 文本                          |
| signText      | string   | 是    |     | 最终参与签名的字符串                             |
| sign          | string   | 是    |     | 生成后的签名值                                |

## 2. 验证签名

### 请求示例

```http
POST /api/security/debug/verify
Content-Type: application/json

{
  "appId": "YWRtaW4wMDAx",
  "signatureType": "A",
  "signFields": ["username", "password"],
  "payloadText": "{\"username\":\"admin\",\"password\":\"123456\"}",
  "requestId": "trace-001",
  "traceId": "trace-001",
  "timestamp": "1700000000",
  "sign": "BASE64_AES_SIGNATURE"
}
```

### 字段说明

| 字段            | 类型       | 是否必须 | 默认值   | 字段解释                                                         |
|---------------|----------|------|-------|--------------------------------------------------------------|
| appId         | string   | 是    |       | 应用 ID                                                        |
| payloadText   | string   | 是    |       | 待验签 JSON 对象文本                                                |
| requestId     | string   | 否    | 自动生成  | 请求链路 ID，参与签名；验签真实请求时必须使用原始 `X-Trace-Id`                      |
| timestamp     | string   | 否    | 当前秒级时间 | 秒级 Unix 时间戳；验签真实请求时必须使用原始 `X-Timestamp`                         |
| signatureType | string   | 否    | R     | 签名方式：`A` AES、`R` RSA                                         |
| signFields    | string[] | 否    | ["*"] | 参与验签字段                                                       |
| sign          | string   | 是    |       | 待验证签名                                                        |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "验证成功",
  "data": {
    "appId": "1",
    "requestId": "trace-001",
    "traceId": "trace-001",
    "timestamp": "1700000000",
    "signatureType": "A",
    "signFields": ["username", "password"],
    "payload": {
      "username": "admin",
      "password": "123456"
    },
    "payloadText": "{\n  \"username\": \"admin\",\n  \"password\": \"123456\"\n}",
    "signText": "password=123456&username=admin&key=<md5(appId-traceId-timestamp)>",
    "sign": "BASE64_AES_SIGNATURE",
    "verified": true
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 验签结果 |

#### data 内部字段

| 字段            | 类型       | 是否必须 | 默认值   | 字段解释                                   |
|---------------|----------|------|-------|----------------------------------------|
| appId         | string   | 是    |       | 实际参与验签的 AppID                          |
| requestId     | string   | 是    |       | 实际参与验签的请求标识                            |
| traceId       | string   | 是    |       | 与 `requestId` 等值，便于前端按 `X-Trace-Id` 回填 |
| timestamp     | string   | 是    |       | 实际参与验签的秒级 Unix 时间戳                    |
| signatureType | string   | 是    |       | 实际使用的签名方式                              |
| signFields    | string[] | 是    |       | 实际参与验签的字段列表                            |
| payload       | object   | 是    |       | 归一化后的待验签对象                             |
| payloadText   | string   | 是    |       | 归一化后的 JSON 文本                          |
| signText      | string   | 是    |       | 最终参与验签的字符串                             |
| sign          | string   | 是    |       | 待验证签名                                  |
| verified      | bool     | 是    | false | 签名是否匹配                                 |

## 3. 加密数据

### 请求示例

```http
POST /api/security/debug/encrypt
Content-Type: application/json

{
  "appId": "YWRtaW4wMDAx",
  "cryptoType": "A",
  "cipherFields": ["password"],
  "payloadText": "{\"username\":\"admin\",\"password\":\"123456\"}"
}
```

### 字段说明

| 字段         | 类型       | 是否必须 | 默认值 | 字段解释                          |
|------------|----------|------|-----|-------------------------------|
| appId      | string   | 是    |     | 应用 ID，和 `X-App-Id` 语义一致       |
| cryptoType | string   | 否    | A   | 加密方式：`A` AES、`R` RSA          |
| cipherFields | string[] | 是    |     | 需要字段级加密的 JSON 字段路径，禁止 `cipher` |
| payloadText | string   | 是    |     | 待处理 JSON 对象文本                  |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "操作成功",
  "data": {
    "appId": "YWRtaW4wMDAx",
    "cryptoType": "A",
    "cipherFields": ["password"],
    "cipherHeader": "WyJwYXNzd29yZCJd",
    "payload": {
      "username": "admin",
      "password": "123456"
    },
    "payloadText": "{\n  \"username\": \"admin\",\n  \"password\": \"123456\"\n}",
    "resultPayload": {
      "username": "admin",
      "password": "base64密文"
    },
    "resultPayloadText": "{\n  \"username\": \"admin\",\n  \"password\": \"base64密文\"\n}"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 加密结果 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| appId             | string   | 是 |  | 实际使用的 AppID |
| cryptoType        | string   | 是 |  | 实际使用的加密方式 |
| cipherFields      | string[] | 是 |  | 实际使用的字段配置 |
| cipherHeader      | string   | 是 |  | 可直接用于 `X-Cipher` 的请求头值 |
| payload           | object   | 是 |  | 输入对象 |
| payloadText       | string   | 是 |  | 输入 JSON 文本 |
| resultPayload     | object   | 是 |  | 字段加密后的对象 |
| resultPayloadText | string   | 是 |  | 字段加密后的 JSON 文本 |
