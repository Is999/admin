# 接口文档访问接口

接口文档页面在生产环境必须先登录后台，再创建文档访问会话。文档首页校验 `docs.index`，具体 Markdown 资源按目录继续校验细分文档权限。

## 1. 创建文档访问会话

### 基本信息

- 请求方式：`POST`
- 请求路径：`/api/docs/session`
- RouteMeta：`docs.index`
- 权限码：`docs.index`
- 审计动作：无
- 鉴权：后台登录 Token

### 请求头

| 字段 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| Authorization | string | 是 | `Bearer <token>` |

### 请求参数

无。

### 响应字段

| 字段 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| expiresIn | int | 是 | 文档会话 cookie 最长有效秒数 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "成功",
  "data": {
    "expiresIn": 3600
  }
}
```

### Cookie 说明

- 名称：`admin_docs_token`
- Path：`/api/docs`
- HttpOnly：是
- SameSite：`Lax`
- Secure：HTTPS 请求或 `X-Forwarded-Proto=https` 时启用

## 2. 访问接口文档

创建会话后，浏览器可直接访问：

```text
GET /api/docs#/
```

文档静态资源支持以下路径：

- `GET /api/docs`
- `GET /api/docs/:path`
- `GET /api/docs/:path/:sub`
- `GET /api/docs/:path/:sub/:file`
- `GET /api/docs/:path/:sub/:group/:file`

生产环境访问这些资源时，后端会优先读取 `Authorization` 请求头；没有请求头时读取文档会话 cookie。

路径以 `api/接口文档` 开头时，Admin 后端会通过 `api_service.internal_base_url` 内网读取 API 服务的 `/internal/docs` 文档资源。API 服务不直接向浏览器提供文档站，且内网资源只放行 API 接口文档，不输出角色文档、安全清单等内部资料。
