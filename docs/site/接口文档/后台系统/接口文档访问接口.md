# 接口文档访问接口

接口文档页面必须先登录后台，再创建文档访问会话。后台接口文档入口校验 `docs.index`，前台 API 文档入口校验 `docs.api_service.index`，具体 Markdown 资源按 `docs.file.*` 单篇文档权限校验。

## 1. 创建文档访问会话

### 基本信息

- 请求方式：`POST`
- 请求路径：`/api/docs/session`
- 后台接口文档 RouteMeta：`docs.index`
- 前台 API 文档 RouteMeta：`docs.api_service.index`
- 权限码：按入口和具体 Markdown 资源匹配 `admin_permission.module`
- 审计动作：无
- 鉴权：后台登录 Token

### 请求头

| 字段 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| Authorization | string | 是 | `Bearer <token>` |

### 请求参数

无。

### 响应字段

| 字段 | 类型 | 是否必须 | 说明 |
|------|------|----------|------|
| expiresIn | int | 是 | 文档会话 cookie 最长有效秒数 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "成功",
  "data": {
    "expiresIn": 3600
  }
}
```

### Cookie 说明

- 名称：`admin_docs_token`
- Path：`/api/docs`
- HttpOnly：是
- SameSite：`Lax`
- Secure：HTTPS 请求或 `X-Forwarded-Proto=https` 时启用

## 2. 访问接口文档

创建会话后，浏览器可直接访问：

```text
GET /api/docs#/
GET /api/docs/api#/接口文档/前台系统/系统接口
```

文档静态资源支持以下路径：

- `GET /api/docs`
- `GET /api/docs/:path`
- `GET /api/docs/:path/:sub`
- `GET /api/docs/:path/:sub/:file`
- `GET /api/docs/:path/:sub/:group/:file`

访问这些资源时，所有运行模式都必须携带有效后台凭证。后端会优先读取 `Authorization` 请求头；没有请求头时读取文档会话 cookie，并按当前管理员权限过滤侧边栏、搜索索引和 Markdown 资源。

`/api/docs/api` 是前台 API 文档独立入口，页面壳由 Admin 提供，侧边栏和 Markdown 内容由 Admin 后端通过 `api_service.internal_base_url` 内网读取 API 服务的 `/internal/docs` 文档资源。API 服务不直接向浏览器提供文档站；内网资源只输出 `docs/site/_sidebar.md`、`docs/site/接口文档/**/*.md` 和 `docs/site/角色文档/**/*.md`，不输出 `route_security_manifest.json` 等非 Markdown 机器资产。

### 权限存储与匹配

- 文档权限不单独建文档表，统一存放在 `admin_permission.module`；入口、目录和单篇 Markdown 文档都有稳定 module。
- 单篇文档权限使用 `docs.file.<文档站相对路径>`，例如 `docs.file.接口文档/后台系统/权限管理接口.md` 和 `docs.file.api/接口文档/前台系统/系统接口.md`。
- 角色授予哪些文档权限存放在 `admin_role_permission_rel`；用户绑定角色存放在 `admin_role_rel`。
- 访问时后端按用户启用角色聚合权限匹配单篇文档 module；目录权限只控制入口和分组，不替代子文件读权限。超级管理员角色拥有全部文档权限。
- 角色保存时会自动补齐权限树祖先和同文档入口权限；补齐父级只保证菜单路径完整，不代表父级可读取全部子文件。
- 侧边栏、顶部导航和搜索索引按同一套权限过滤；直接访问 Markdown 资源仍会再次校验，避免只隐藏菜单但直链可读。

单篇文档权限由 `internal/database/assets/document_permission_seed.sql` 维护，`internal/routealias/docs.go` 的 `DocsContentPaths` 与实际 Markdown 文件列表有回归测试校验，避免新增文档后漏配权限表。新增文档权限不得拆出独立 `document_permission_*.sql` 迁移；新增 `INSERT IGNORE` 行按自增 `id` 递增追加，已上线环境由发布单手动执行并刷新权限缓存。
目录权限用于文档入口、侧边栏分组和权限树路径，具体 Markdown 文件仍必须授予对应 `docs.file.*` 权限。
