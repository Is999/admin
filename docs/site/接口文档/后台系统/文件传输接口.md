# 文件传输接口

---

## 1. 初始化上传会话

### 请求示例

```http
POST /api/file-transfer/upload/init
Content-Type: application/json

{
  "fileName": "admin.xlsx",
  "fileSize": 1048576,
  "contentType": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
  "bizType": "admin_export",
  "chunkSize": 524288,
  "checksum": "sha256-xxx"
}
```

### 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| fileName | string | 是 |  | 文件名 |
| fileSize | int64 | 是 |  | 文件大小，单位字节 |
| contentType | string | 否 |  | 文件 MIME 类型 |
| bizType | string | 否 | default | 业务类型，用于选择上传策略 |
| chunkSize | int64 | 否 | 系统默认值 | 分片大小，单位字节 |
| checksum | string | 否 |  | 文件完整性摘要 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "创建成功",
  "data": {
    "uploadId": "upload_xxx",
    "chunkSize": 524288,
    "chunkTotal": 2,
    "expireAt": "2026-05-11T12:00:00+08:00"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 上传会话信息 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| uploadId | string | 是 |  | 上传会话 ID |
| chunkSize | int64 | 是 |  | 后端确认的分片大小 |
| chunkTotal | int | 是 |  | 分片总数 |
| expireAt | string | 是 |  | 会话过期时间 |

## 2. 查询上传状态

### 请求示例

```http
GET /api/file-transfer/upload/status?uploadId=upload_xxx
```

### 查询参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| uploadId | string | 是 |  | 上传会话 ID |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "uploadId": "upload_xxx",
    "uploadedChunks": [0],
    "completed": false,
    "accessUrl": ""
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 上传状态 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| uploadId | string | 是 |  | 上传会话 ID |
| uploadedChunks | int[] | 是 | [] | 已上传分片序号 |
| completed | bool | 是 | false | 是否已完成 |
| accessUrl | string | 否 |  | 完成后可访问地址 |

## 3. 上传分片

### 请求示例

```http
PUT /api/file-transfer/upload/chunk?uploadId=upload_xxx&chunkIndex=0
Content-Type: application/octet-stream

<binary>
```

### 查询参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| uploadId | string | 是 |  | 上传会话 ID |
| chunkIndex | int | 是 |  | 分片序号，从 0 开始 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "上传成功",
  "data": {
    "uploadId": "upload_xxx",
    "chunkIndex": 0,
    "uploaded": true
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 分片上传结果 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| uploadId | string | 是 |  | 上传会话 ID |
| chunkIndex | int | 是 | 0 | 分片序号 |
| uploaded | bool | 是 | false | 当前分片是否已保存 |
