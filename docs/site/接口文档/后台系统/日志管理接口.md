# 日志管理接口

---

## 1. 查询后台操作日志

### 请求示例

```http
GET /api/admin-logs?page=1&pageSize=10&traceID=trace-001&username=admin
```

### 查询参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| page | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页数量 |
| traceID | string | 否 |  | Trace ID |
| userID | int | 否 |  | 操作用户 ID |
| username | string | 否 |  | 操作用户名 |
| action | string | 否 |  | 操作动作 |
| startTime | string | 否 |  | 开始时间，格式 `YYYY-MM-DD HH:MM:SS` |
| endTime | string | 否 |  | 结束时间，格式 `YYYY-MM-DD HH:MM:SS` |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "id": 1,
        "userID": 1,
        "username": "admin",
        "action": "query",
        "route": "admin.list",
        "method": "GET",
        "describe": "查询管理员列表",
        "data": "{}",
        "ip": "127.0.0.1",
        "ipaddr": "本机",
        "traceId": "trace-001",
        "spanId": "span-001",
        "httpStatus": 200,
        "bizCode": 1,
        "latencyMs": 12,
        "success": true,
        "errorMessage": "",
        "createdAt": "2026-05-11 12:00:00"
      }
    ],
    "total": 1
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 日志分页数据 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| list | array | 是 | [] | 日志列表 |
| total | int | 是 | 0 | 总数 |

#### list[] 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 日志 ID |
| userID | int | 是 | 0 | 操作用户 ID |
| username | string | 否 |  | 操作用户名 |
| action | string | 是 |  | 操作动作 |
| route | string | 是 |  | 路由别名，例如 `task.workflow.trigger` |
| method | string | 是 |  | HTTP 方法 |
| describe | string | 否 |  | 中文业务说明 |
| data | string | 否 |  | 脱敏后的请求/响应摘要 |
| ip | string | 否 |  | 客户端 IP |
| ipaddr | string | 否 |  | IP 归属地 |
| traceId | string | 否 |  | 请求链路 Trace ID |
| spanId | string | 否 |  | 当前请求 Span ID |
| httpStatus | int | 是 | 0 | HTTP 状态码 |
| bizCode | int | 是 | 0 | 业务状态码 |
| latencyMs | int64 | 是 | 0 | 请求耗时，单位毫秒 |
| success | bool | 是 | false | 请求是否成功 |
| errorMessage | string | 否 |  | 失败摘要 |
| createdAt | string | 是 |  | 创建时间 |

### 使用说明

- `traceId + spanId` 是日志、监控、问题单和 Trace 平台的联查主键。
- `route/action/describe` 由统一路由元数据产出，前后端不要各自维护第二套中文说明。
- `data` 为脱敏摘要，不保证包含全部原始敏感字段。
