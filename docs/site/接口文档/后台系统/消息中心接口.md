# 消息中心接口

消息中心用于承载后台管理员的站内信与通知提醒，支持：

- 管理员收件箱分页查询、筛选与关键字检索
- 顶部铃铛通知列表（用于前端 Header 通知弹层）
- 未读数量统计
- 标记已读（批量/全部）
- 删除消息（软删除，批量/清空全部已读）
- 发送消息（指定收件人/广播全部启用管理员）

消息中心属于个人消息能力，只校验登录 token，不配置 `admin_permission` 权限码。

> 统一响应结构、鉴权、签名与加密规则请先阅读：[基础规范与认证接口](接口文档/后台系统/基础规范与认证接口.md)

---

## 接口总览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/admin-messages` | 查询管理员消息收件箱（分页/筛选） |
| `GET` | `/api/admin-messages/sent` | 查询管理员已发送消息（分页/筛选） |
| `GET` | `/api/admin-messages/{id}/receivers` | 查询消息收件人已读明细（仅发送人可见） |
| `GET` | `/api/admin-messages/unread-count` | 查询未读消息数量 |
| `GET` | `/api/admin-messages/notifications` | 查询顶部铃铛通知列表 |
| `PATCH` | `/api/admin-messages/read` | 标记消息已读（批量/全部） |
| `POST` | `/api/admin-messages/delete` | 删除消息（软删除，批量/清空全部已读） |
| `POST` | `/api/admin-messages/send` | 发送消息（站内信/通知） |
| `POST` | `/api/admin-messages/handle` | 标记消息已处理（仅可处理消息类型支持） |

---

## 字段约定

### 消息类型（type）

常用消息类型建议保持统一编码，便于筛选与告警联动：

- `admin_login`：管理员登录
- `system_notice`：系统通知
- `security_alert`：安全告警
- `task_result`：任务结果
- `task_exception`：任务异常
- `approval_notice`：审批提醒
- `leave_message`：他人留言
- `work_handover`：工作交接

### 消息等级（level）

- `1`：info（信息）
- `2`：warning（警告）
- `3`：error（错误）

## 查询收件箱（分页/筛选）

```http
GET /api/admin-messages?page=1&pageSize=10&readStatus=0&type=admin_login&keyword=登录
```

查询参数（均可选）：

- `page`：页码（默认 1）
- `pageSize`：每页条数（默认 10）
- `type`：消息类型
- `level`：消息等级（1/2/3）
- `readStatus`：已读状态（0 未读，1 已读）
- `keyword`：标题/内容关键字（模糊匹配）
- `startTime` / `endTime`：时间范围（格式 `YYYY-MM-DD HH:MM:SS`）

响应 `data` 示例：

```json
{
  "list": [
    {
      "id": 10001,
      "type": "admin_login",
      "level": 1,
      "title": "管理员登录提醒",
      "content": "管理员[super999]于 2026-05-05 12:00:00 登录后台，登录IP：127.0.0.1",
      "data": "{\"ip\":\"127.0.0.1\",\"username\":\"super999\"}",
      "link": "",
      "senderAdminId": 1,
      "senderAdminName": "super999",
      "handledStatus": 0,
      "handledByAdminName": "",
      "handledAt": "",
      "isRead": false,
      "readAt": "",
      "createdAt": "2026-05-05 12:00:00"
    }
  ],
  "total": 1
}
```

## 查询通知列表（顶部铃铛）

```http
GET /api/admin-messages/notifications?limit=10
```

## 标记已读（批量/全部）

```http
PATCH /api/admin-messages/read
```

批量标记：

```json
{
  "ids": [10001, 10002]
}
```

标记全部：

```json
{
  "all": true
}
```

响应 `data` 示例：

```json
{
  "affected": 2
}
```

---

## 删除消息（软删除，批量/清空已读）

```http
POST /api/admin-messages/delete
```

批量删除：

```json
{
  "ids": [10001, 10002]
}
```

清空全部已读：

```json
{
  "allRead": true
}
```

响应 `data` 示例：

```json
{
  "affected": 10
}
```

---

## 发送消息（站内信/通知）

```http
POST /api/admin-messages/send
```

广播给全部启用管理员（不传 `receiverIDs` 即广播）：

```json
{
  "type": "work_handover",
  "level": 1,
  "title": "交接提醒",
  "content": "今日 18:00 前完成交接",
  "data": "{\"deadline\":\"2026-05-05 18:00:00\"}",
  "link": "/work/handover"
}
```

发送给指定管理员：

```json
{
  "type": "leave_message",
  "level": 1,
  "title": "有人留言",
  "content": "请查看工单 #12345",
  "receiverIDs": [2, 3]
}
```

## 标记已处理（需人工处理的消息）

> 仅 `security_alert` / `task_exception` / `approval_notice` 这类需要人工介入的消息支持标记；其它类型调用会返回失败结果。

```http
POST /api/admin-messages/handle
```

请求示例：

```json
{
  "id": 10020
}
```

响应 `data` 示例（被他人抢先处理时也返回 success，并通过 `alreadyHandled` 提示）：

```json
{
  "id": 10020,
  "handledStatus": 1,
  "handledByAdminName": "admin03",
  "handledAt": "2026-05-05 12:20:00",
  "alreadyHandled": true
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 消息列表、未读数量、操作结果或处理结果 |

#### 消息 data.list[] 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 消息 ID |
| type | string | 是 |  | 消息类型 |
| level | int | 是 | 1 | 消息等级 |
| title | string | 是 |  | 标题 |
| content | string | 是 |  | 内容 |
| link | string | 否 |  | 跳转链接 |
| senderAdminId | int | 否 | 0 | 发送管理员 ID |
| senderAdminName | string | 否 |  | 发送管理员名称 |
| handledStatus | int | 是 | 0 | 处理状态 |
| isRead | bool | 否 | false | 是否已读 |
| createdAt | string | 是 |  | 创建时间 |
