# 秘钥管理接口

## 接口总览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/secret-keys` | 查询秘钥列表 |
| `POST` | `/api/secret-keys` | 新增秘钥 |
| `PATCH` | `/api/secret-keys/:id` | 编辑秘钥 |
| `PATCH` | `/api/secret-keys/status/:id` | 启用或停用秘钥 |
| `POST` | `/api/secret-keys/cache/refresh/:uuid` | 刷新秘钥缓存 |
| `POST` | `/api/secret-keys/validate` | 预检秘钥路径与材料可用性 |
| `POST` | `/api/secret-keys/self-check/:uuid` | 执行运行态自检并刷新缓存 |

## 秘钥保存请求

```json
{
  "uuid": "1",
  "title": "管理后台前端应用",
  "keyVersion": "v1",
  "stableVersion": "v1",
  "grayVersion": "v2",
  "grayPercent": 10,
  "aesKeyRef": "/etc/admin-cron/keys/1/v1/aes_key",
  "aesIvRef": "/etc/admin-cron/keys/1/v1/aes_iv",
  "rsaPublicKeyUserRef": "/etc/admin-cron/keys/1/v1/user_public.pem",
  "rsaPrivateKeyServerRef": "/etc/admin-cron/keys/1/v1/server_private.pem",
  "status": 0,
  "signStatus": 1,
  "cryptoStatus": 1,
  "versionStatus": 1,
  "remark": "部署校验通过后启用"
}
```

> **注意**：当前项目的秘钥管理已经升级为“主配置 + 版本配置”模式。`secret_key` 主表管理稳定版本、灰度版本、灰度比例、应用状态，以及 `signStatus` / `cryptoStatus` 两个运行时安全开关；具体 AES/RSA 材料全部落在 `secret_key_version` 表。系统只允许保存文件绝对路径，不再接受 AES 明文或 RSA PEM 文本。新增、编辑、切换状态、刷新缓存与运行态自检均属于敏感操作，使用 `MFAScenarioSecretKeyManage = 11`；后台配置需要 MFA 时必须提交 `twoStepKey`、`twoStepValue`。
>
> - `signStatus=1` 时，当前 AppID 才会执行请求验签和响应回签
> - `cryptoStatus=1` 时，当前 AppID 才会执行请求解密和响应加密
> - 关闭某一链路后，不再强制要求对应材料参与启用校验
> - `rsaPublicKeyServerRef` 为兼容字段，通常不配置；为空时由服务端私钥派生公钥

## 预检接口

用于在保存前检查路径格式、文件是否存在、AES 长度、RSA PEM 格式以及服务端密钥是否可用。

```json
POST /api/secret-keys/validate
{
  "uuid": "1",
  "title": "管理后台前端应用",
  "keyVersion": "v2",
  "stableVersion": "v1",
  "grayVersion": "v2",
  "grayPercent": 20,
  "aesKeyRef": "/etc/admin-cron/keys/1/v2/aes_key",
  "aesIvRef": "/etc/admin-cron/keys/1/v2/aes_iv",
  "rsaPublicKeyUserRef": "/etc/admin-cron/keys/1/v2/user_public.pem",
  "rsaPrivateKeyServerRef": "/etc/admin-cron/keys/1/v2/server_private.pem",
  "status": 1,
  "signStatus": 1,
  "cryptoStatus": 1,
  "versionStatus": 1,
  "twoStepKey": "xxx",
  "twoStepValue": "xxx"
}
```

返回结果会包含 `allPassed`、`canSave`、`canEnable` 与 `items[]` 分项结果；前端只展示校验结果，不返回文件真实内容和内部错误链。

## 运行态自检接口

用于对已落库 `uuid` 执行缓存刷新、AES 加解密、RSA 签名验签、RSA 加解密链路检查。

```json
POST /api/secret-keys/self-check/YWRtaW4wMDAx
{
  "keyVersion": "v1",
  "twoStepKey": "xxx",
  "twoStepValue": "xxx"
}
```

返回结果会额外包含：

- `runtimeChecked=true`
- `cacheRefreshed=true/false`
- `durationMs`
- `items[]` 中的运行态分项结果，如 `AES加解密自检`、`RSA验签自检`

## 运行时缓存与灰度

- 路由缓存：`secret_key_route:{uuid}`
- 版本材料缓存：`secret_key_aes:{uuid}:{keyVersion}`、`secret_key_rsa:{uuid}:{keyVersion}`
- 版本材料索引：`secret_key_version:index:{uuid}`，成员为当前 AppID 下已写入的 AES/RSA 版本材料真实 key；刷新和删除按索引精确清理，不使用前缀 `SCAN`
- 未显式指定版本时，运行时先命中 `stableVersion`
- 显式传入的 `X-Key-Version` 最长 64 位，不能包含空白字符或通配符；不存在的随机版本不会写入空值缓存，避免被恶意制造高基数 Redis Key
- 当 `grayVersion` 非空且 `grayPercent > 0` 时，按 `X-Gray-Key` 进行稳定分桶灰度
- 服务端会把最终命中的版本回写到响应头 `X-Key-Version`

## RSA 工具说明

秘钥管理页的 RSA 工具已调整为前端本地生成与下载，不再提供服务端 `generate/download` 接口。运维需要的用户公钥和服务端私钥应在浏览器本地生成后离线分发到挂载目录，再由秘钥配置引用对应文件绝对路径。

## 统一响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 秘钥列表、秘钥详情、预检结果或自检结果 |

### 秘钥 data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 秘钥 ID |
| uuid | string | 是 |  | 应用标识 |
| title | string | 是 |  | 应用名称 |
| stableVersion | string | 否 |  | 稳定版本 |
| grayVersion | string | 否 |  | 灰度版本 |
| grayPercent | int | 否 | 0 | 灰度比例 |
| signStatus | int | 是 | 0 | 签名状态 |
| cryptoStatus | int | 是 | 0 | 加密状态 |
| status | int | 是 | 0 | 应用状态 |
