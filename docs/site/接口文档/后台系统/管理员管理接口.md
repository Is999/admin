# 管理员管理接口

## 用户列表

`GET /api/admins`

查询参数：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `page` | number | 页码 |
| `pageSize` | number | 每页数量 |
| `username` | string | 登录用户名 |
| `realName` | string | 真实姓名 |
| `status` | number | `1` 启用，`0` 禁用 |
| `roleID` | number | 角色 ID |

响应 `data`：

```json
{
  "list": [
    {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "email": "",
      "phone": "",
      "status": 1,
      "mfaStatus": 0,
      "roleIDs": [1],
      "roles": []
    }
  ],
  "total": 1
}
```

## 用户详情

`GET /api/admins/:id`

## 新增用户

`POST /api/admins`

请求：

```json
{
  "username": "editor",
  "realName": "编辑员",
  "password": "123456",
  "email": "editor@example.com",
  "phone": "13800000000",
  "avatar": "",
  "description": "",
  "roleIDs": [2],
  "mfaSecureKey": "",
  "twoStepKey": "",
  "twoStepValue": ""
}
```

登录用户名已存在时返回：

```json
{
  "status": false,
  "code": 100002,
  "message": "用户[editor]已存在"
}
```

## 编辑用户

`PATCH /api/admins/:id`

请求：

```json
{
  "realName": "编辑员",
  "email": "editor@example.com",
  "phone": "13800000000",
  "avatar": "/avatars/editor.png",
  "description": "运营编辑员",
  "password": "",
  "roleIDs": [2],
  "isUpdateRoles": true,
  "twoStepKey": "",
  "twoStepValue": ""
}
```

`password` 为空或不传时不修改密码。`roleIDs` 是数组，沿用轻量签名策略不参与字段签名；需要同步角色时同时传 `isUpdateRoles=true`。

规则补充：

- 仅允许修改自己可管理范围内的管理员。
- 该接口仅用于基础资料和角色编辑，不再承载 `status`、`mfaStatus`、`mfaSecureKey` 等专用变更。
- 若请求中带非空 `password`，后台按 `MFAScenarioChangePassword` 场景校验；否则按 `MFAScenarioEditUser` 场景校验。

## 用户状态、密码、角色

| 方法 | 路径 | 请求体 | 说明 |
| --- | --- | --- | --- |
| `DELETE` | `/api/admins/:id` | `{"twoStepKey":"","twoStepValue":""}` | 删除用户，使用 `MFAScenarioDeleteUser` |
| `PATCH` | `/api/admins/status/:id` | `{"status":1,"twoStepKey":"","twoStepValue":""}` | 修改用户状态，使用 `MFAScenarioUserStatus` |
| `POST` | `/api/admins/password/reset/:id` | `{"password":"123456","twoStepKey":"","twoStepValue":""}` | 重置密码，使用 `MFAScenarioResetUserPassword` |
| `POST` | `/api/admins/initial-state/reset/:id` | `{"password":"123456","twoStepKey":"","twoStepValue":""}` | 重置首次状态，使用 `MFAScenarioResetUserInitialState` |
| `GET` | `/api/admins/roles/:id` | - | 查询用户角色 |
| `PATCH` | `/api/admins/roles/:id` | `{"roleIDs":[1,2],"twoStepKey":"","twoStepValue":""}` | 覆盖保存用户角色，使用 `MFAScenarioEditUser` |
| `GET` | `/api/admins/mfa-secret-key-url/:id` | - | 为未启用或秘钥异常的管理员生成一次性 MFA 绑定二维码 |
| `PATCH` | `/api/admins/mfa-status/:id` | `{"mfaStatus":1,"twoStepKey":"","twoStepValue":""}` | 修改管理员 MFA 状态；启用后若账号未绑定设备，下次登录会先完成 MFA 绑定 |
| `POST` | `/api/admins/exports` | `{"username":"admin","status":1}` | 创建管理员导出任务 |
| `GET` | `/api/admins/exports/status/:jobId` | - | 查询管理员导出任务状态 |
| `GET` | `/api/admins/exports/download/:jobId` | - | 下载管理员导出文件 |

MFA 绑定地址包含 TOTP 秘钥，仅对未启用或秘钥异常的账号生成，并通过响应字段级加密返回；已启用且秘钥可用的账号禁止通过该接口读回现有秘钥。

管理员资料、状态、密码、角色或 MFA 状态变更先提交数据库，再失效目标账号登录态和展示缓存。每次受保护请求都以主库角色与权限关系作为最终授权依据，因此 Redis 失效失败不会延长已撤销权限；缓存同步失败时仍返回成功业务码，并通过 `data.syncPending=true` 提示手动刷新展示缓存，不要重复提交原变更请求。

### 管理员导出请求示例

```http
POST /api/admins/exports
Content-Type: application/json

{
  "username": "admin",
  "realName": "管理员",
  "roleID": 1,
  "status": 1
}
```

### 管理员导出响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "成功",
  "data": {
    "jobId": "5a1e2448-2d08-4dbe-a17c-8a143f3026de",
    "taskId": "task_xxx",
    "queue": "maintenance",
    "status": "queued"
  }
}
```

导出任务在后台 worker 中按同一筛选条件统计总量并生成文件，提交接口不执行大结果集查询。轮询 `GET /api/admins/exports/status/:jobId` 返回：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | `queued/running/succeeded/failed` |
| `progress` | int | 总量统计和导出阶段为 `0-99`，文件持久化成功后为 `100` |
| `processed` | int64 | 已处理行数 |
| `total` | int64 | worker 开始时按导出条件统计，完成后校正为实际导出行数 |
| `averageRowsPerSec` | int64 | 任务全局平均导出速度（行/秒） |
| `estimatedSeconds` | int64 | 按全局平均速度计算的预计剩余秒数；总量统计阶段和完成后为 `0` |
| `downloadReady` | bool | 导出对象已持久化且可下载时为 `true` |

限制说明：

- 不允许修改自己的账号状态。
- 不允许对自己的账号执行后台重置密码、重置首次状态、删除账号。
- 上述接口统一受角色范围限制，只能操作自己可管理范围内的管理员。
- 新增用户、编辑用户和覆盖保存用户角色时，`roleIDs` 只能提交当前管理员可分配的下级角色；普通管理员不能分配自身最高角色、上级角色、同级角色、其它分支角色或禁用角色。
- 后台代操作是否要求 MFA，统一由 `adminMFACheckEnable` 与 `adminDisableMFACheckScenario` 决定。
- 角色分配接口的 `roleIDs` 为数组字段，不进入轻量签名载荷；`twoStepKey`、`twoStepValue` 仍按安全策略参与请求签名。
- 新增用户的登录用户名必须唯一，重复时返回业务码 `100002`。

## 统一响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 业务响应数据，列表接口返回分页对象，详情接口返回管理员对象，操作接口可为空 |

### 管理员 data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 管理员 ID |
| username | string | 是 |  | 登录用户名 |
| realName | string | 否 |  | 真实姓名 |
| email | string | 否 |  | 邮箱 |
| phone | string | 否 |  | 手机号 |
| status | int | 是 | 1 | 状态，1 启用，0 禁用 |
| mfaStatus | int | 是 | 0 | MFA 状态 |
| roleIDs | int[] | 否 | [] | 绑定角色 ID |
| roles | array | 否 | [] | 绑定角色对象 |
| jobId | string | 否 |  | 导出任务 ID，仅导出接口返回 |
