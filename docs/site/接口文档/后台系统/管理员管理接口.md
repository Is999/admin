# 管理员管理接口

## 用户列表

`GET /api/admins`

查询参数：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `page` | number | 页码 |
| `pageSize` | number | 每页数量 |
| `username` | string | 登录用户名 |
| `realName` | string | 真实姓名 |
| `status` | number | `1` 启用，`0` 禁用 |
| `roleID` | number | 角色 ID |

响应 `data`：

```json
{
  "list": [
    {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "email": "",
      "phone": "",
      "status": 1,
      "mfaStatus": 0,
      "roleIDs": [1],
      "roles": []
    }
  ],
  "total": 1
}
```

## 用户详情

`GET /api/admins/:id`

## 新增用户

`POST /api/admins`

请求：

```json
{
  "username": "editor",
  "name": "editor",
  "realName": "编辑员",
  "password": "123456",
  "email": "editor@example.com",
  "phone": "13800000000",
  "avatar": "",
  "description": "",
  "roleIDs": [2]
}
```

## 编辑用户

`PATCH /api/admins/:id`

请求字段同新增；`password` 为空或不传时不修改密码。需要同步角色时传 `roleIDs` 和 `isUpdateRoles=true`。

规则补充：

- 仅允许修改自己可管理范围内的管理员。
- 该接口仅用于基础资料和角色编辑，不再承载 `status`、`mfaStatus`、`mfaSecureKey` 等专用变更。
- 若请求中带非空 `password`，后台按 `MFAScenarioChangePassword` 场景校验；否则按 `MFAScenarioEditUser` 场景校验。

## 用户状态、密码、角色

| 方法 | 路径 | 请求体 | 说明 |
| --- | --- | --- | --- |
| `DELETE` | `/api/admins/:id` | `{"twoStepKey":"","twoStepValue":""}` | 删除用户，使用 `MFAScenarioDeleteUser` |
| `PATCH` | `/api/admins/status/:id` | `{"status":1,"twoStepKey":"","twoStepValue":""}` | 修改用户状态，使用 `MFAScenarioUserStatus` |
| `POST` | `/api/admins/password/reset/:id` | `{"password":"123456","twoStepKey":"","twoStepValue":""}` | 重置密码，使用 `MFAScenarioResetUserPassword` |
| `POST` | `/api/admins/initial-state/reset/:id` | `{"password":"123456","twoStepKey":"","twoStepValue":""}` | 重置首次状态，使用 `MFAScenarioResetUserInitialState` |
| `GET` | `/api/admins/roles/:id` | - | 查询用户角色 |
| `PATCH` | `/api/admins/roles/:id` | `{"roleIDs":[1,2]}` | 覆盖保存用户角色 |
| `GET` | `/api/admins/mfa-secret-key-url/:id` | - | 查询管理员 MFA 绑定二维码 |
| `PATCH` | `/api/admins/mfa-status/:id` | `{"mfaStatus":1,"twoStepKey":"","twoStepValue":""}` | 修改管理员 MFA 状态 |
| `POST` | `/api/admins/exports` | `{"ids":[1],"columns":["username"]}` | 创建管理员导出任务 |
| `GET` | `/api/admins/exports/status/:jobId` | - | 查询管理员导出任务状态 |
| `GET` | `/api/admins/exports/download/:jobId` | - | 下载管理员导出文件 |

### 管理员导出请求示例

```http
POST /api/admins/exports
Content-Type: application/json

{
  "ids": [1, 2],
  "columns": ["username", "realName", "email"],
  "fileName": "admins.xlsx"
}
```

### 管理员导出响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "任务已创建",
  "data": {
    "jobId": "export_xxx",
    "status": "pending"
  }
}
```

限制说明：

- 不允许修改自己的账号状态。
- 不允许对自己的账号执行后台重置密码、重置首次状态、删除账号。
- 上述接口统一受角色范围限制，只能操作自己可管理范围内的管理员。
- 后台代操作是否要求 MFA，统一由 `adminMFACheckEnable` 与 `adminDisableMFACheckScenario` 决定。

## 统一响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 业务响应数据，列表接口返回分页对象，详情接口返回管理员对象，操作接口可为空 |

### 管理员 data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 管理员 ID |
| username | string | 是 |  | 登录用户名 |
| realName | string | 否 |  | 真实姓名 |
| email | string | 否 |  | 邮箱 |
| phone | string | 否 |  | 手机号 |
| status | int | 是 | 1 | 状态，1 启用，0 禁用 |
| mfaStatus | int | 是 | 0 | MFA 状态 |
| roleIDs | int[] | 否 | [] | 绑定角色 ID |
| roles | array | 否 | [] | 绑定角色对象 |
| jobId | string | 否 |  | 导出任务 ID，仅导出接口返回 |
