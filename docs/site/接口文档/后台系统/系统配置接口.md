# 系统常量配置接口

`sys_config` 表归属 config 库，查询、新增、编辑和缓存回源均使用配置库连接，避免误写后台管理库。

## 接口总览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/dicts` | 查询配置列表 |
| `POST` | `/api/dicts` | 新增配置 |
| `PATCH` | `/api/dicts/:id` | 编辑配置 |
| `GET` | `/api/dicts/cache/:uuid` | 查看配置缓存 |
| `POST` | `/api/dicts/cache/refresh/:uuid` | 刷新配置缓存 |
| `GET` | `/api/dicts/export` | 导出配置 Excel |
| `POST` | `/api/dicts/import` | 导入配置 Excel |

## 配置保存请求

```json
{
  "uuid": "adminMFACheckEnable",
  "title": "后台MFA强校验开关",
  "type": 6,
  "value": true,
  "example": true,
  "remark": "true 表示登录和敏感场景都强制进入 MFA 规则",
  "page": "security",
  "pid": 6
}
```

**配置类型说明**：
- `0` 分组
- `1` 对象
- `2` 数组
- `3` 字符串
- `4` 整数
- `5` 浮点数
- `6` 布尔

## 当前安全相关配置项

### MFA 相关字典

| 字典标识 | 类型 | 示例值 | 说明 |
| --- | --- | --- | --- |
| `adminMFACheckEnable` | `6` 布尔 | `true` | 后台系统强制开启 MFA 校验开关；开启后，登录校验和后台代操作敏感场景都会进入 MFA 规则 |
| `adminMFACheckFrequency` | `4` 整数 | `300` | MFA 二次校验频率，单位秒；在频率窗口内可复用最近一次场景票据，前端同一标签页会按票据过期时间自动复用，超时后需要重新校验 |
| `adminDisableMFACheckScenario` | `1` 对象 | `{"13":"前台用户管理","14":"API运行态热加载"}` | 禁用 MFA 二次校验的场景集合；对象 key 为场景值，登录场景不受此配置影响 |

### 场景值说明

| 场景值  | 场景常量                               | 说明           |
|------|------------------------------------|--------------|
| `0`  | `MFAScenarioLogin`                 | 登录后校验        |
| `1`  | `MFAScenarioChangePassword`        | 修改密码         |
| `2`  | `MFAScenarioStatus`                | 修改 MFA 状态    |
| `3`  | `MFAScenarioSecret`                | 修改 MFA 秘钥    |
| `4`  | `MFAScenarioUserStatus`            | 修改管理员状态      |
| `5`  | `MFAScenarioAddUser`               | 新增管理员        |
| `6`  | `MFAScenarioEditUser`              | 编辑管理员资料/角色   |
| `7`  | `MFAScenarioResetUserPassword`     | 后台重置管理员密码    |
| `8`  | `MFAScenarioResetUserInitialState` | 后台重置管理员首次状态  |
| `9`  | `MFAScenarioDeleteUser`            | 后台删除管理员      |
| `10` | `MFAScenarioUserTagLeaseRelease`   | 释放用户标签工作流互斥锁 |
| `11` | `MFAScenarioSecretKeyManage`       | 秘钥管理敏感操作     |
| `12` | `MFAScenarioRuntimeConfigManage`   | 运行配置发布/回滚/导入 |
| `13` | `MFAScenarioUserManage`         | 前台用户管理敏感操作   |
| `14` | `MFAScenarioAPIRuntimeManage`      | API 运行态热加载      |

### 规则补充

- 个人中心修改自己密码且已提供旧密码时，不额外要求 MFA 二次票据。
- 个人中心修改自己的 MFA 状态、MFA 秘钥时，必须先完成对应场景校验。
- 后台代操作是否要求 MFA，统一以 `adminMFACheckEnable` 和 `adminDisableMFACheckScenario` 为准。
- 后台重置密码、重置首次状态、删除管理员均已使用独立场景值，可单独配置禁用校验。
- 详细业务规则见 [多因素认证与管理员角色规则](角色文档/后端开发/多因素认证与管理员角色规则.md)。

## 统一响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 配置列表、配置详情、导入结果或缓存刷新结果 |

### 配置 data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 配置 ID |
| uuid | string | 是 |  | 配置标识 |
| title | string | 是 |  | 配置标题 |
| type | int | 是 |  | 配置类型 |
| value | any | 否 |  | 配置值 |
| page | string | 否 |  | 所属页面 |
| pid | int | 否 | 0 | 父级配置 ID |
