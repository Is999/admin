# 缓存管理接口

## 接口总览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/caches` | 查询内置缓存目标 |
| `GET` | `/api/caches/server-info` | 查询 Redis 服务信息 |
| `GET` | `/api/caches/key-info?key=xxx` | 查询 Redis Key 详情 |
| `GET` | `/api/caches/keys?key=xxx` | 搜索 Redis Key |
| `GET` | `/api/caches/key-info/search?key=xxx` | 查询搜索结果 Key 详情 |
| `POST` | `/api/caches/refresh` | 刷新指定缓存 |
| `POST` | `/api/caches/refresh-all` | 刷新全部内置缓存 |
| `POST` | `/api/caches/warmup` | 按模板 Key 预热缓存（从无到有构建实例 Key） |

## 刷新指定缓存请求

```json
{
  "key": "admin:role:tree",
  "type": "hash"
}
```

## 搜索 Key 示例

```http
GET /api/caches/keys?key=admin:info:*&page=1&pageSize=20
```

可选诊断参数：

- `source`：请求来源标记，便于日志排障。当前前端约定：
  - `manual_search`
  - `route_jump`
  - `template_locate`
  - `template_drawer`
- `page`：页码，从 `1` 开始，默认 `1`。
- `pageSize`：每页返回数量，默认 `20`，最大 `50`。

> **说明**：`key` 至少需要 3 个非通配符字符。后端最多在受控窗口内确认 `5000` 个真实 Key，并按 `page/pageSize` 只返回当前页；详情接口会对
`secret_key_*`、`admin:info:*`、MFA 票据等敏感缓存脱敏。
>
> **搜索约束**：
>
> - 允许：精确 Key、已登记模板 Key 搜索，例如 `permission_tree`、`admin:info:*`、`config_uuid:*`
> - 禁止：未知通配符搜索，例如 `*permission*`、`?role*`、`permission*`，避免触发 Redis 高成本全库扫描
> - 模板 Key 搜索优先走后端模板 provider 枚举候选实例，再批量校验 Redis 是否存在，不依赖通用 `SCAN MATCH`

## 按模板预热缓存

### 场景说明

缓存管理页支持“刷新缓存”，但刷新前提是 Redis 中已存在对应的实例 Key。
当业务侧尚未触发过某类缓存（例如模板 Key 形如 `config_uuid:{uuid}`），仅靠刷新无法“从无到有”构建缓存实例。

`/api/caches/warmup` 用于把**模板 Key**扩展为**真实实例 Key 列表**并逐个触发刷新，从而完成缓存预热。

### 请求

`POST /api/caches/warmup`

```json
{
  "templateKey": "config_uuid:{uuid}",
  "limit": 2000
}
```

字段说明：

- `templateKey`：模板 Key（必填）。仅允许后端白名单内的模板，避免误用导致全库扫描。
- `limit`：单次预热最大实例数量（可选）。用于限制本次枚举与刷新规模；不传时使用后端默认值。

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "成功",
  "data": {
    "templateKey": "config_uuid:{uuid}",
    "total": 128,
    "success": 126,
    "failed": 2,
    "failedKeys": [
      "config_uuid:9e8d1d16-acde-4b9a-acde-0c3b3d7d3f8b"
    ],
    "latencyMs": 182
  }
}
```

### 注意事项

- 该接口属于运维能力，建议在低峰期执行，并优先通过 `limit` 控制单次规模。
- 若存在失败项，返回会采样部分失败 key 供排障；失败任务不会影响其他 key 的预热执行。

## 统一响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 缓存目标列表、Redis Key 详情、搜索结果或刷新结果 |

### 预热 data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| templateKey | string | 是 |  | 模板 Key |
| total | int | 是 | 0 | 实例 Key 总数 |
| success | int | 是 | 0 | 成功数量 |
| failed | int | 是 | 0 | 失败数量 |
| failedKeys | string[] | 否 | [] | 采样失败 Key |
| latencyMs | int64 | 是 | 0 | 执行耗时，单位毫秒 |
