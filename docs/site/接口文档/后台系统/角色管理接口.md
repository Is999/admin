# 角色管理接口

---

## 1. 查询角色列表

### 请求示例

```http
GET /api/roles?page=1&pageSize=10&title=运营&status=1
```

### 查询参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| page | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页数量 |
| title | string | 否 |  | 角色名称 |
| status | int | 否 |  | 状态，1 启用，0 禁用 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "id": 2,
        "title": "运营",
        "pid": 0,
        "status": 1,
        "describe": "运营角色"
      }
    ],
    "total": 1
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 角色分页数据 |

#### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| list | array | 是 | [] | 角色列表 |
| total | int | 是 | 0 | 总数 |

## 4. 新增角色

### 请求示例

```http
POST /api/roles
Content-Type: application/json

{
  "title": "运营",
  "pid": 0,
  "status": 1,
  "describe": "运营角色",
  "permissions": [1, 2, 3]
}
```

### 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| title | string | 是 |  | 角色名称 |
| pid | int | 否 | 0 | 父级角色 ID |
| status | int | 是 | 1 | 状态，1 启用，0 禁用 |
| describe | string | 否 |  | 角色说明 |
| permissions | int[] | 否 | [] | 绑定权限 ID 列表 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "创建成功",
  "data": {
    "id": 2
  }
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | object | 否 |  | 新增角色 ID 对象 |

### 重复名称响应

```json
{
  "status": false,
  "code": 30001,
  "message": "角色[运营]已存在"
}
```

## 8. 查询角色权限树

> 安全边界：`isPid=y` 仅用于查询可作为上级的角色权限范围；普通管理员不能因上级为超级管理员而获得全量可选权限。

### 请求示例

```http
GET /api/roles/permissions/tree/2/n
```

路由模板：`GET /api/roles/permissions/tree/:id/:isPid`

### 路径参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 角色 ID |
| isPid | string | 是 | n | 是否按父级维度查询，`y/n` |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": []
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |
| data | array | 是 | [] | 权限树 |

## 9. 覆盖保存角色权限

> 安全边界：超级管理员角色权限只读；只有超级管理员在父级为顶级/超级管理员时可选择全部权限，普通管理员只能配置自己角色的后代角色。保存时会自动补齐已选权限的祖先目录/菜单权限，并丢弃祖先链不完整的半截授权，避免子菜单有权限但父菜单不可见。

### 请求示例

```http
PATCH /api/roles/permissions/2
Content-Type: application/json

{
  "permissions": [1, 2, 3]
}
```

路由模板：`PATCH /api/roles/permissions/:id`

### 路径参数

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| id | int | 是 |  | 角色 ID |

### 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| permissions | int[] | 是 | [] | 权限 ID 列表 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "保存成功"
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 业务状态码 |
| message | string | 否 |  | 消息 |

### 高危操作说明

- 角色名称必须唯一，新增或编辑重复名称时返回业务码 `30001`。
- 禁用角色时会级联禁用全部下级角色，并清理相关缓存。
- 删除角色时会级联软删除全部下级角色，同时清理管理员角色关系和角色权限关系。
