# 接口文档统一规范

本文档定义 `admin-cron` API 文档的统一书写样板。所有具体接口文档均按“请求示例、字段说明、响应示例、响应字段说明、data 内部字段”的结构维护，便于前端、测试、后端和运维按同一格式联调。

> **AI 开发必读**：AI 新增或修改接口文档前，必须先阅读 [AI开发规范](../角色文档/后端开发/AI开发规范.md)，并同步检查业务码、多语言文案、路由元数据和测试。

---

## 1. 通用响应结构

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 0 | 业务状态码，不同业务错误返回不同状态码，便于定位问题 |
| message | string | 否 |  | 当前语言下的响应消息，受 `Accept-Language` 影响 |
| data | object/array/null | 否 |  | 业务响应数据，不同接口结构不同 |

---

## 2. 常用业务状态码

| 状态码 | 状态含义 | 典型场景 |
|------|----------|----------|
| 1 | 成功 | 请求处理成功 |
| 2 | 失败 | 通用业务失败 |
| 3 | 参数错误 | 参数缺失、格式错误、业务约束不满足 |
| 4 | 未登录或登录过期 | Token 缺失、无效或过期 |
| 5 | 无权限 | 没有访问当前接口或数据范围的权限 |
| 6 | 需要 MFA 校验 | 登录或敏感操作需要动态码 |
| 7 | 需要修改密码 | 首次登录或管理员重置后必须改密 |
| 8 | MFA 校验过期 | 二次校验票据失效，需要重新校验 |
| 9 | 需要绑定 MFA | 系统强制启用 MFA，但账号尚未绑定 |

## 3. 通用请求头

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| Authorization | string | 条件必填 |  | 登录态接口必填，格式 `Bearer <token>` |
| Content-Type | string | 否 | application/json | JSON 请求体建议固定为 `application/json` |
| Accept-Language | string | 否 | zh-CN | 国际化语言，支持中文和英文 |
| X-Trace-Id | string | 否 |  | 请求链路追踪 ID，建议前端生成并透传 |
| X-App-Id | string | 条件必填 |  | 开启签名或加密时必填，值为 `base64(appID)` |
| X-Signature | string | 条件必填 |  | 请求签名方式：`M` MD5、`A` AES、`R` RSA |
| X-Crypto | string | 条件必填 |  | 加密方式：`A` AES、`R` RSA |
| X-Cipher | string | 条件必填 |  | 加密字段，`cipher` 表示整体请求体加密 |
| X-Key-Version | string | 否 |  | 指定秘钥版本，最长 64 位，不允许空白和通配符 |
| X-Gray-Key | string | 否 |  | 灰度秘钥路由分桶键 |

## 4. 字段说明表规范

请求字段、查询参数、路径参数、响应字段统一使用以下表头：

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| 示例字段 | string | 是 |  | 示例说明 |

路径参数没有默认值时可省略默认值列，但具体接口优先保持和样板一致。

## 5. 分页响应规范

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "查询成功",
  "data": {
    "list": [],
    "total": 0
  }
}
```

### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| list | array | 是 | [] | 当前页列表 |
| total | int | 是 | 0 | 符合条件的总数 |
| meta | object | 否 |  | 附加统计或分页元信息 |

---

## 6. 链路追踪与排障

- 所有接口建议携带 `X-Trace-Id`，后端会将其写入访问日志、业务日志、审计日志和 Trace 上下文。
- 任务类接口响应中的 `workflowId`、`taskId` 可用于继续查询任务状态。
- 分片类任务排障时优先结合 `workflow_id`、`mode`、`node`、`shard` 等链路字段定位。
