# 用户标签内网接口

## 1. 适用场景

供 admin 等内网服务调用，免 JWT，但要求来源 IP 命中内网白名单。

## 2. 内网工作流触发接口

### 接口

```http
POST /internal/user-tags/workflows
```

### 请求示例

```http
POST /internal/user-tags/workflows
Content-Type: application/json

{
  "mode": "targeted",
  "tagTypes": [2, 3],
  "uids": ["10001", "10002"],
  "dryRun": true
}
```

### 说明

- 请求体与 `POST /api/user-tags/workflows` 完全一致。
- 适合 admin 传 `uids + tagTypes` 发起 `targeted` 骨架任务；`uids` 必须使用十进制字符串，避免跨语言大整数精度丢失。

## 3. 内网指定标签重算接口

### 接口

```http
POST /internal/user-tags/recalculations
```

### 请求示例

```http
POST /internal/user-tags/recalculations
Content-Type: application/json

{
  "tag_types": [30, 31],
  "queue": "maintenance",
  "dry_run": true
}
```

### 说明

- 请求体与 `POST /api/user-tags/recalculations` 完全一致。
- 适合内网服务直接发起 `recalculate` 骨架任务。

## 4. 注意事项

- 这两类接口虽然免 JWT，但不是公网接口。
- 如果从公网环境调用，会被内网中间件直接拒绝。

## 5. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 响应状态码 |
| message | string | 否 |  | 消息 |
| data | object | 否 |  | 返回结构与对应公网接口一致 |
