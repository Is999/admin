# 指定标签重算接口

## 1. 适用菜单

- 后台菜单：`用户标签`
- 页面第二块表单：`指定标签重算`

## 2. 公网接口

### 接口

```http
POST /api/user-tags/recalculations
```

## 3. 语义说明

该接口保留 admin 历史字段，内部转换成 `recalculate` 工作流处理。当前版本只触发骨架链路，`tag_types` 只作为不透明标签类型编号参与参数校验和去重。

## 4. 请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `tag_types` | int[] | 是 | 指定标签类型，兼容 admin 历史字段 |
| `tagTypes` | int[] | 否 | 驼峰兼容字段 |
| `queue` | string | 否 | 指定队列 |
| `shard_total` / `shardTotal` | int | 否 | 分片数 |
| `batch_size` / `batchSize` | int | 否 | 批次大小 |
| `worker_count` / `workerCount` | int | 否 | Worker 数 |
| `dry_run` / `dryRun` | bool | 否 | 只计算不落库 |
| `unique_ttl_seconds` / `uniqueTTLSeconds` | int | 否 | 去重 TTL 秒 |
| `timeout_seconds` / `timeoutSeconds` | int | 否 | 超时秒数 |

## 5. 请求示例

```json
{
  "tag_types": [30, 31],
  "queue": "maintenance",
  "shard_total": 10,
  "batch_size": 2000,
  "worker_count": 4,
  "dry_run": false,
  "unique_ttl_seconds": 1800,
  "timeout_seconds": 1800
}
```

## 6. 成功响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "触发成功",
  "data": {
    "message": "已启动 2 个标签重算任务",
    "tag_count": 2,
    "taskId": "task_xxx",
    "workflowId": "wf_xxx",
    "workflowName": "user_tag.tag.recalculate",
    "queue": "maintenance"
  }
}
```

## 7. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 响应状态码 |
| message | string | 否 |  | 消息 |
| data | object | 是 |  | 指定标签重算触发结果 |

### data 内部字段

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| message | string | 是 |  | 任务启动说明 |
| tag_count | int | 是 | 0 | 本次重算标签数量 |
| taskId | string | 是 |  | 任务 ID |
| workflowId | string | 是 |  | 工作流实例 ID |
| workflowName | string | 是 |  | 工作流名称 |
| queue | string | 是 |  | 任务队列 |

## 8. 使用建议

- 用于验证 `recalculate` 入口、参数归一化、去重和任务投递。
- 如果只想确认链路可触发，可以先开 `dry_run=true`。
