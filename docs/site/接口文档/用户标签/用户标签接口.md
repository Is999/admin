# 用户标签接口总览

## 1. 文档说明

用户标签接口当前服务于框架骨架，负责验证工作流编排和释放互斥锁。默认实现不内置具体标签规则，主工作流只接受 `dryRun=true`；非 dry-run 请求会在任何清表、切表操作前失败关闭。

- “用户标签工作流接口”负责 `full / delta / targeted / recalculate` 模式触发。
- “指定标签重算接口”负责触发指定标签重算工作流。
- “内网接口”负责 admin 等内网服务免鉴权调用。

## 3. 模式速览

| 模式 | 说明 |
| --- | --- |
| `full` | 全量骨架链路，包含结果表准备和提交。 |
| `delta` | 增量骨架链路，复用主 DAG。 |
| `targeted` | 指定 UID 的骨架链路，要求传入 `uids` 和 `tagTypes`。 |
| `recalculate` | 指定 `tagTypes` 的骨架链路。 |

## 4. 响应字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| status | bool | 是 | false | 状态: true 成功, false 失败 |
| code | int | 是 | 1 | 响应状态码 |
| message | string | 否 |  | 消息 |
| data | object/array/null | 否 |  | 用户标签工作流触发结果或重算结果 |
