# MFA 与管理员角色规则

## 适用范围

- 本文约束后台管理员登录态 MFA 校验、个人中心 MFA 绑定与启停、管理员管理敏感操作、以及父角色可操作边界。
- 系统配置项统一使用 `sys_config` 字典中的 `adminMFACheckEnable`、`adminMFACheckFrequency`、`adminDisableMFACheckScenario`。

## MFA 绑定与启用规则

- `mfa_status=0` 时，进入绑定/启用页必须携带本次新秘钥。
- 未启用账号校验动态码时，后端先使用本次新秘钥校验。
- 若新秘钥校验失败，且库内仍存在旧 `mfa_secure_key`，再回退用旧秘钥校验。
- 哪个秘钥校验通过，启用时就落库哪个秘钥，并同步把 `mfa_status` 更新为 `1`。
- 两套秘钥都校验失败时，统一返回动态码错误。
- 关闭 MFA 时只关闭 `mfa_status`，不主动清理库内旧秘钥。
- 已启用账号不走自助换绑；个人中心只允许未启用账号继续绑定或更新待启用秘钥。

## 登录规则

- 场景 `0` 为 `MFAScenarioLogin`。
- 若 `adminMFACheckEnable=1`，登录后必须完成 MFA 校验。
- 若账号自身 `mfa_status=1`，即使系统未强制开启，登录后也必须完成 MFA 校验。
- `need_reset_password=1` 时，优先放行改密流程，改密成功后再回到 MFA 规则。

## 登录态拦截与业务码约定

鉴权中间件按顺序执行：账号状态/IP → 必须改密 → 登录态 MFA → 业务权限。为了避免交互逻辑打架，登录态 MFA 的业务码拆分为两类：

| 业务码 | 常量 | 语义 | 触发条件（后端） | 前端动作 |
| --- | --- | --- | --- | --- |
| `6` | `CheckMFACode` | 需要校验 MFA 动态码（已绑定/已启用账号） | 需要登录态 MFA 且未通过登录态校验标记 | 弹出“输入动态码”校验，成功后重放原请求 |
| `9` | `CheckMFABind` | 需要先绑定并启用 MFA（强制开启且未启用账号） | `adminMFACheckEnable=1` 且 `mfa_status!=1` | 进入“绑定引导（二维码+动态码）”，成功后重放原请求 |
| `7` | `CheckPasswordReset` | 必须先修改登录密码 | `need_reset_password=1` 且当前路由不在放行白名单 | 跳转个人中心强制改密页 |
| `8` | `CheckMFAAgain` | MFA 二次确认票据已过期/缺失 | 敏感操作场景票据失效 | 页面侧拉起二次校验弹窗并重试提交 |

> 约束：当 `adminMFACheckEnable=1` 且 `mfa_status!=1` 时，登录态拦截必须返回 `CheckMFABind(9)`，不能再返回 `CheckMFACode(6)`，否则前端会误进入“只能输动态码”的页面，导致无法完成首次绑定。

## 交互流程图（登录 → 改密 → MFA → 初始化）

```mermaid
flowchart TD
  A[登录提交 /api/auth/login] --> B{needResetPassword==1?}
  B -- 是 --> PR[跳转个人中心强制改密] --> PR2[改密成功后回到登录后初始化]
  B -- 否 --> C{adminMFACheckEnable==1 且 mfaStatus!=1?}

  C -- 是 --> D[进入 MFA 绑定引导\n(展示二维码/秘钥 + 输入动态码)]
  D --> E[/api/admin/checkMfaSecure\nscenarios=0/]
  E --> F{动态码校验通过?}
  F -- 否 --> D
  F -- 是 --> G[/api/admin/checkMfaSecure\nscenarios=2/]
  G --> H[/api/admin/updateMfaStatus\nmfaStatus=1 + twoStep/]
  H --> I[标记登录态 MFA 已完成]
  I --> J[并发初始化\n/api/login/after/info + /api/auth/codes]

  C -- 否 --> K{需要登录态 MFA?}
  K -- 否 --> J
  K -- 是 --> L[输入动态码校验]
  L --> E2[/api/admin/checkMfaSecure\nscenarios=0/]
  E2 --> M{动态码校验通过?}
  M -- 否 --> L
  M -- 是 --> I
```

## 十二个场景规则

| 场景值  | 场景常量                               | 规则                                                  |
|------|------------------------------------|-----------------------------------------------------|
| `0`  | `MFAScenarioLogin`                 | 登录后校验；系统强制开启或账号已启用 MFA 时必须校验                        |
| `1`  | `MFAScenarioChangePassword`        | 修改他人密码按系统强制开关判断；个人中心修改自己密码只要提供当前旧密码并校验通过，不再额外要求 MFA |
| `2`  | `MFAScenarioStatus`                | 修改自己的 MFA 状态必须校验；修改他人的 MFA 状态只在系统强制开启时要求校验          |
| `3`  | `MFAScenarioSecret`                | 修改自己的 MFA 秘钥必须校验；票据不能与秘钥管理场景串用                      |
| `4`  | `MFAScenarioUserStatus`            | 自己的账号状态禁止修改；修改他人状态只在系统强制开启时要求校验                     |
| `5`  | `MFAScenarioAddUser`               | 新增管理员只在系统强制开启时要求校验                                  |
| `6`  | `MFAScenarioEditUser`              | 编辑管理员基础资料/角色只在系统强制开启时要求校验                           |
| `7`  | `MFAScenarioResetUserPassword`     | 后台重置他人密码只在系统强制开启时要求校验；不允许重置自己的密码                    |
| `8`  | `MFAScenarioResetUserInitialState` | 后台重置他人首次状态只在系统强制开启时要求校验；不允许重置自己的首次状态                |
| `9`  | `MFAScenarioDeleteUser`            | 后台删除管理员只在系统强制开启时要求校验；不允许删除自己的账号                     |
| `10` | `MFAScenarioUserTagLeaseRelease`   | 释放用户标签工作流互斥锁只在系统强制开启时要求校验                           |
| `11` | `MFAScenarioSecretKeyManage`       | 秘钥管理保存、修改、状态切换等敏感操作只在系统强制开启时要求校验；票据不能与 MFA 秘钥场景串用 |

## 个人中心规则

- `/api/admin/updateMine` 仅更新当前登录管理员自己的基础资料，不做 MFA 二次校验。
- `/api/admin/updatePassword` 为“自己改自己密码”入口；已提供旧密码时，不额外要求 MFA 二次校验。
- `/api/admin/updateMfaStatus` 为“自己改自己 MFA 状态”入口：
  - 启用时先走绑定校验，再签发二次票据完成正式启用。
  - 关闭时必须先完成 `MFAScenarioStatus` 场景校验。
- `/api/admin/updateMfaSecureKey` 为“自己改自己 MFA 秘钥”入口，必须先完成 `MFAScenarioSecret` 场景校验。

## 管理员管理规则

- 父角色只能修改自己可管理范围内的管理员。
- 普通管理员的“可管理范围”定义为：自己拥有的角色及其全部子孙角色。
- 超级管理员可管理全部管理员角色范围。
- 父角色可以修改子角色管理员，以及自己的基础资料。
- 自己的账号状态不能修改。
- 自己的账号不能执行后台重置密码、重置首次状态、删除账号。
- 编辑管理员基础资料接口不再承载 `status`、`mfa_status`、`mfa_secure_key` 这类专用字段。
- 管理员状态变更必须走专用状态接口。
- 管理员 MFA 状态变更必须走专用 MFA 状态接口。
- 管理员密码重置、首次状态重置、删除账号，同样受角色范围限制，并统一按各自独立 MFA 场景校验。

## 校验频率与禁用场景

- `adminMFACheckFrequency` 表示二次校验票据的有效窗口，单位秒。
- 在有效窗口内，同一管理员的普通敏感操作可复用最近一次已签发的 `twoStepKey/twoStepValue`，前端同一标签页会按票据过期时间自动复用。
- `MFA 状态启用/关闭` 场景依赖本次动态码校验的秘钥来源元信息，仍要求使用当前场景票据，不跨场景复用。
- 超过频率窗口后，敏感操作需要重新完成 MFA 动态码校验。
- 二次校验票据写入 `admin:mfa:two_step:{adminID}:{ticketKey}`，同时写入 `admin:mfa:two_step:index:{adminID}` 精确索引；账号初始化和后台重置首次状态按索引删除，不使用 `SCAN admin:mfa:two_step:{adminID}:*`。
- `adminDisableMFACheckScenario` 表示禁用 MFA 二次校验的场景对象，key 为场景值。
- 登录场景 `0` 不受 `adminDisableMFACheckScenario` 影响。
- 后台“代操作”场景是否需要 MFA，统一只看系统强制开关和禁用场景配置。

## 前端联调要求

- 个人资料修改不弹 MFA。
- 个人中心修改自己密码不弹 MFA。
- 个人中心修改自己的 MFA 秘钥时，弹窗校验必须携带本次新秘钥。
- 管理员新增、编辑、修改他人状态、修改他人 MFA 状态、重置密码、重置首次状态、删除账号等代操作，前端是否预弹 MFA 统一跟随 `adminMFACheckEnable`。
