# 库表路由规范

当前项目默认只使用一个 MySQL 主库，统一通过 `svc.DatabaseMain` 访问。`site_mysql` 仅作为后续拆库扩展入口，默认不配置。

## 默认模型

- `mysql`：必填主库配置。
- `site_mysql`：可选命名扩展库配置，按 `site_mysql.<name>` 注册。
- `ReadDB(database)`：返回只读连接，空值或 `main` 指向主库。
- `WriteDB(database)`：返回写连接，空值或 `main` 指向主库。

## 扩展示例

确需拆库时，新增一个有明确用途的命名库：

```yaml
site_mysql:
  archive:
    write_data_source: "user:password@tcp(mysql:3306)/admin_archive?charset=utf8mb4&parseTime=true&loc=Local"
    read_data_sources: []
    max_open_conns: 20
    max_idle_conns: 10
    conn_max_lifetime: 300
    debug: false
```

业务配置需要使用扩展库时，显式填写对应名称，例如 `archive_jobs[].database: "archive"`。未配置的名称不会自动回退，启动或执行阶段应尽早暴露错误。

## 调用规则

- 默认使用 `svc.DatabaseMain`。
- 不在业务逻辑中手写 DSN 或临时打开 MySQL 连接。
- 新增扩展库前先确认数据边界、连接池规模、读写分离和健康检查诉求。
- 扩展库名称只做前后空白整理；命名应短而明确，便于运维识别。
