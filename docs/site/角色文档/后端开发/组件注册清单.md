# 组件注册清单

本文档记录 admin 默认组件、路由模块、任务插件和运行时扩展注册入口。新增默认能力时优先修改对应规格，避免注册入口散落在 main、bootstrap、handler 或业务逻辑中。

## 统一入口

| 注册类型 | 统一入口 | 说明 |
| --- | --- | --- |
| 启动组件 | `internal/bootstrap/components/builtin/specs.go:DefaultSpecs` | 统一声明通用收集器、任务运行时、HTTP 服务组件顺序，并派生真实组件和清单 |
| 启动组件执行器 | `internal/bootstrap/components:NewRegistry` | 通用执行器负责顺序注册和名称唯一校验 |
| HTTP 路由 | `internal/handler/routes.go:builtinRouteModuleSpecs` + 各模块 `RouteSpecs` | 统一声明内置路由模块顺序和路由规格来源，真实注册、`RouteContract` 和注册清单从规格派生 |
| 任务插件 | `internal/bootstrap/components/builtin/task.go:DefaultTaskPluginSpecs` + `internal/jobs/plugins.go:PluginSpecs` | 统一声明核心、业务和维护任务插件顺序，任务组件和注册清单从规格派生 |
| 运行时扩展 | 各包 `RuntimeRegistrySpecs` | `pkg/storage`、`internal/logic/file`、`internal/infra/collectorx` 分别声明自身注册入口，`bootstrap` 统一汇总清单 |

## 单库数据库路由

| 场景 | 统一入口 | 说明 |
| --- | --- | --- |
| 默认物理库 | `mysql` 配置 | 当前唯一必填 MySQL 入口 |
| 主库读连接 | `internal/svc/dbrouter.go:ServiceContext.ReadDB` | 空值或 `main` 指向主库只读连接 |
| 主库写连接 | `internal/svc/dbrouter.go:ServiceContext.WriteDB` | 空值或 `main` 指向主库写连接 |
| 拆库扩展 | `site_mysql.<name>` | 可选命名扩展库，默认不配置 |
| 归档控制库 | `internal/jobs/archive/service.go:WithControlDatabase` | 默认使用主库 |
| 归档源库 | `internal/jobs/archive/service.go:jobSourceWriteDB` | 由 `archive_jobs[].database` 指定，默认 `main` |

更完整的约束、接入步骤和禁止项，见 `库表路由规范.md`。

默认所有表都使用同一个 `mysql` 物理库；需要拆库时通过 `site_mysql.<name>` 注册命名扩展库。

## 外部运行期配置

| 场景     | 配置入口                   | 说明                                                                                                            |
|--------|------------------------|---------------------------------------------------------------------------------------------------------------|
| 运行期小配置/兼容种子 | `config_files.runtime` | `workflows` 仍由文件管理；`task_periodic`、`archive_jobs` 在 DB 模式下仅作为手动导入或草稿为空时的兼容种子 |

主配置只保留 `task` 基础参数和 `scheduler`、`archive` 基础参数；`task.periodic`、`archive.jobs` 必须通过运行配置草稿发布，顶层 `user_tag` 迁移到运行期配置文件的 `workflows.user_tag`。

首次导入种子文件格式：

```yaml
task_periodic:
  - name: "user-tag-delta-daily"
    cron: "0 3 * * *"
    workflow: "user_tag.delta.refresh"
  - name: "cache-refresh-every-10s"
    cron: "*/10 * * * * *"
    workflow: "cache.refresh"
  - name: "user-tag-runtime-cleanup"
    every_seconds: 21600
    workflow: "user_tag.runtime.cleanup"
  - name: "user-tag-event-outbox-retry-scan"
    every_seconds: 600
    workflow: "user_tag.event_outbox.retry_scan"

archive_jobs:
  - name: "admin_log"
    enabled: true
    database: "main"
    table_name: "admin_log"
    archive_condition: ""
    delete_condition: ""
    archive_delay_days: 2
    archive_window_seconds: 3600
    delete_delay_days: 30
    delete_window_seconds: 3600

workflows:
  user_tag:
    enabled: true
    default_shard_total: 1
    runtime_shard_total: 1024
    result_shard_total: 1

```

文件模式热加载会把主配置和 `config_files.runtime` 指向的外部文件一起计算指纹；数据库模式下 `task_periodic/archive_jobs` 以 active release 为准。

本地运行时从项目根目录执行 `go run ./cmd/admin -f etc/config.yaml -mode 7`，`config_files.runtime: "config.d/runtime.yaml"` 会解析到 `etc/config.d/runtime.yaml`。

任务插件启停、handler 注册和 workflow 定义注册属于启动期能力。热加载可以刷新批次、分片、限速等运行参数，但不能把启动时未注册的插件在线补注册；遇到 `restartRequired=true` 且 `restartReason` 包含插件启停项时，需要重启对应进程。

## 扩展规则

1. 新增默认启动组件：在 `builtin.DefaultSpecs` 增加规格，由 `builtin.DefaultComponents` 和 `manifest.Default()` 自动派生。
2. 新增默认路由模块：先在对应模块维护 `RouteSpecs`，再在 `internal/handler/routes.go:builtinRouteModuleSpecs` 增加规格来源；真实注册、`DefaultRouteContracts` 和注册清单会从规格派生。
3. 新增默认任务插件：先在 `internal/jobs/<业务>/task` 定义 `Setup(...)`，再加入 `internal/jobs/plugins.go:PluginSpecs`；启动链和注册清单会从规格派生。
4. 新增 Collector bizType：通过 `Collector.RegisterProcessor` 或 `Collector.RegisterProcessorFunc` 注册批量消费逻辑。
5. 新增文件上传业务类型：通过 `file.RegisterFileUploadPolicy` 注册，并保证扩展名、大小、可见性都有明确策略。
6. 新增默认运行时扩展入口：在能力归属包维护 `RuntimeRegistrySpecs`，`bootstrap` 直接从这些规格派生注册清单。
7. 调整表归属库：优先判断是否仍可留在默认 `mysql`；确需拆库时补 `site_mysql.<name>` 并通过 `ReadDB` / `WriteDB` 访问对应名称。
8. 所有默认注册名称必须唯一；启动装配和 `go test ./internal/bootstrap ./internal/handler` 会校验真实注册集合的名称冲突。
