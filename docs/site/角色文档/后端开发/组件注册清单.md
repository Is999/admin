# 组件注册清单

本文档记录 admin-cron 默认组件、路由模块、任务插件和运行时扩展注册入口。新增组件时优先修改 `internal/bootstrap/registrations.go`，避免注册入口散落在 main、bootstrap、handler 或业务逻辑中。

## 统一入口

| 注册类型 | 统一入口 | 说明 |
| --- | --- | --- |
| 启动组件 | `internal/bootstrap/registrations.go:defaultComponents` | 统一声明通用收集器、任务运行时、HTTP 服务组件顺序 |
| HTTP 路由 | `internal/bootstrap/registrations.go:defaultRouteModules` | 统一声明内置路由模块顺序，HTTP 组件通过 `resolveRouteModules` 合并外部模块 |
| 任务插件 | `internal/bootstrap/registrations.go:defaultTaskPlugins` | 统一声明内置任务插件顺序，任务组件通过 `resolveTaskPlugins` 合并外部插件 |
| 运行时扩展 | `internal/bootstrap/registrations.go:defaultRuntimeRegistryNames` | 统一记录对象存储、病毒扫描、上传策略和 Collector Processor 注册入口 |
| 注册清单校验 | `internal/bootstrap/registrations.go:ValidateDefaultRegistrationManifest` | 启动前校验清单完整性、名称唯一性和真实默认注册集合一致性 |

## 单库数据库路由

| 场景 | 统一入口 | 说明 |
| --- | --- | --- |
| 默认物理库 | `mysql` 配置 | 当前唯一必填 MySQL 入口 |
| 主库读连接 | `internal/svc/dbrouter.go:ServiceContext.ReadDB` | 空值或 `main` 指向主库只读连接 |
| 主库写连接 | `internal/svc/dbrouter.go:ServiceContext.WriteDB` | 空值或 `main` 指向主库写连接 |
| 拆库扩展 | `site_mysql.<name>` | 可选命名扩展库，默认不配置 |
| 归档控制库 | `internal/jobs/archive/service.go:WithControlDatabase` | 默认使用主库 |
| 归档源库 | `internal/jobs/archive/service.go:jobSourceWriteDB` | 由 `archive_jobs[].database` 指定，默认 `main` |

更完整的约束、接入步骤和禁止项，见 `库表路由规范.md`。

默认所有表都使用同一个 `mysql` 物理库；需要拆库时通过 `site_mysql.<name>` 注册命名扩展库。

当前默认归属：

| 表                                                                           | 默认数据库    | 说明                                                                                           |
|-----------------------------------------------------------------------------|----------|----------------------------------------------------------------------------------------------|
| `archive_segment` / `archive_watermark`                                     | `main`   | 归档控制表                                                                                        |
| `collector_outbox`                                                          | `main`   | Collector 可靠投递兜底表                                                                            |
| `sys_config`                                                                | `main`   | 系统配置表                                                                                        |
| `secret_key` / `secret_key_version`                                         | `main`   | 接口安全秘钥配置表                                                                                    |
| `admin_message` / `admin_message_receiver` / `admin_log`                    | `main`   | 管理员消息和审计日志表                                                                                  |
| `user_tag_*` / `user_tag_sync_*` / `user_tag_runtime_uid` / `user_tag_kafka_outbox` | `main` | 用户标签骨架运行期表 |

## 外部运行期配置

| 场景     | 配置入口                   | 说明                                                                                                            |
|--------|------------------------|---------------------------------------------------------------------------------------------------------------|
| 运行期大配置 | `config_files.runtime` | 唯一外部运行期配置入口，一个文件同时承载 `task_periodic`、`archive_jobs`、`workflows` |

主配置只保留 `task` 基础参数和 `scheduler`、`archive` 基础参数；`task.periodic`、`archive.jobs`、顶层 `user_tag` 必须迁移到运行期配置文件的新结构。

运行期配置文件格式：

```yaml
task_periodic:
  - name: "user-tag-delta-daily"
    cron: "0 3 * * *"
    workflow: "user_tag.delta.refresh"
  - name: "cache-refresh-every-10s"
    cron: "*/10 * * * * *"
    workflow: "cache.refresh"
  - name: "user-tag-runtime-cleanup"
    every_seconds: 21600
    workflow: "user_tag.runtime.cleanup"
  - name: "user-tag-kafka-outbox-retry-scan"
    every_seconds: 600
    workflow: "user_tag.kafka_outbox.retry_scan"

archive_jobs:
  - name: "admin_log"
    enabled: true
    database: "main"
    table_name: "admin_log"
    archive_condition: ""
    delete_condition: ""
    archive_delay_days: 2
    archive_window_seconds: 3600
    delete_delay_days: 30
    delete_window_seconds: 3600

workflows:
  user_tag:
    enabled: true
    default_shard_total: 10

```

热加载会把主配置和 `config_files.runtime` 指向的外部文件一起计算指纹；K8s ConfigMap 原子替换任一文件都会触发重载。

本地运行时从项目根目录执行 `go run . -f etc/config.yaml -mode 7`，`config_files.runtime: "config.d/runtime.yaml"` 会解析到 `etc/config.d/runtime.yaml`。

任务插件启停、handler 注册和 workflow 定义注册属于启动期能力。热加载可以刷新批次、分片、限速等运行参数，但不能把启动时未注册的插件在线补注册；遇到 `restartRequired=true` 且 `restartReason` 包含插件启停项时，需要重启对应进程。

## 扩展规则

1. 新增默认启动组件：只改 `internal/bootstrap/registrations.go:defaultComponents` 和 `DefaultRegistrationManifest`。
2. 新增默认路由模块：先在 `internal/handler/routes.go` 增加 `NewXxxRouteModule`，再加入 `defaultRouteModules`。
3. 新增默认任务插件：先在 `internal/taskruntime` 增加 `NewXxxPlugin`，再加入 `defaultTaskPlugins`。
4. 新增 Collector bizType：通过 `Collector.RegisterProcessor` 或 `Collector.RegisterProcessorFunc` 注册批量消费逻辑。
5. 新增文件上传业务类型：通过 `logic.RegisterFileUploadPolicy` 注册，并保证扩展名、大小、可见性都有明确策略。
6. 新增默认运行时扩展入口：同步 `DefaultRegistrationManifest` 和 `defaultRuntimeRegistryNames`。
7. 调整表归属库：优先判断是否仍可留在默认 `mysql`；确需拆库时补 `site_mysql.<name>` 并通过 `ReadDB` / `WriteDB` 访问对应名称。
8. 所有默认注册名称必须唯一；`go test ./internal/bootstrap` 会校验清单和真实注册集合一致。
