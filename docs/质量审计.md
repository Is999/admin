# admin 重构防漂移与质量审计

审计日期：2026-05-11

## 结论

本次重构不是目录迁移，而是围绕 go-zero 分层、组件注册、任务插件、链路日志和安全策略做了结构化升级。对照 `ark-admin` 后，后台核心能力未发现接口缺失；新项目额外补充了生产探针 `/api/live` 与 `/api/ready`，用于区分存活检查和依赖就绪检查。

## 功能防漂移

- HTTP 路由：旧项目接口集合在新项目中无缺失；新增 `GET /api/live`、`GET /api/ready`。
- 启动模式：保留 `API / Worker / Scheduler` 位掩码独立启动，并通过 `internal/bootstrap` 统一装配。
- 启动错误：配置读取和运行模式解析错误通过 `Wire / WireWithConfigMode` 返回，交给入口顶层统一日志收口，避免装配阶段 panic。
- 路由模块：保留 `RouteModule` 插件化机制，默认模块由规格派生，启动装配只校验真实模块名称冲突。
- 任务插件：保留 `taskruntime` 插件注册，内置 `core / archive / admin_export / user_tag`。
- Collector：保留 Kafka-only 正常链路、失败账本租约领取、重试和死信。
- 任务系统：保留唯一键幂等、DAG workflow、分片、队列暂停恢复、配置热加载。
- 用户标签：保留 UID 分片、规则编译、批量写入、运行态 UID 表和 ClickHouse 聚合。
- 安全：保留 JWT、MFA、签名验签、加解密、密钥版本、IP 白名单和 RBAC。

## 质量提升

- 目录布局：以 `common / helper / internal / pkg / docs` 聚合公共能力、HTTP 辅助、业务实现、复用包和文档。
- 生命周期：`internal/bootstrap` 统一组件注册、启动和停止，避免入口文件散落依赖初始化。
- 进程可靠性：启动装配或服务启动失败时入口返回非零退出码，便于 systemd/Kubernetes 等托管平台及时拉起新实例。
- 错误链路：全项目 `errors.Tag` 使用量已超过 1000 处，错误向上返回并在顶层统一打印。
- 日志链路：统一 `loggerx`，日志字段自动携带 `trace_id / span_id / route / user_id / workflow_id / workflow_node / mode / node / shard / shard_index / shard_total`。
- 数据访问：业务层优先 GORM 链式；原生 SQL 仅保留在动态 DDL、历史归档、批量切表、ClickHouse 聚合等场景。
- Redis 高基数安全：MFA 二次票据和秘钥版本材料缓存均通过索引集合精确清理，缓存管理未知通配搜索直接拒绝，不再依赖生产 Redis `SCAN` 兜底。
- 多语言：响应码通过 `common/i18n` 输出中英文文案，状态码按业务域分段。
- 可观测性：保留 OpenTelemetry、Prometheus 指标、结构化日志和审计日志。
